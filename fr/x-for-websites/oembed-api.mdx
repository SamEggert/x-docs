Vous pouvez utiliser l’API oEmbed pour récupérer par programmation du contenu intégrable, comme des [Tweets](https://developer.x.com/en/docs/twitter-for-websites/embedded-tweets/overview) et des [chronologies](https://developer.x.com/en/docs/twitter-for-websites/timelines/overview).

La réponse de l’API oEmbed renvoie un extrait HTML qui sera automatiquement reconnu lorsque [le JavaScript des widgets X est inclus dans la page](https://developer.x.com/web/javascript/loading).

Veuillez noter que cette API est recommandée pour traiter des opérations en lot ; nous vous conseillons d’utiliser notre outil robuste [publish.x.com](https://publish.x.com/#) pour intégrer du contenu.

<Tabs>
  <Tab title="Embedded timelines">
    L’extrait HTML renvoyé sera automatiquement reconnu comme une [timeline intégrée](https://developer.x.com/en/docs/twitter-for-websites/timelines/overview) lorsque [le JavaScript du widget X est inclus sur la page](https://developer.x.com/web/javascript/loading).

    Le point de terminaison oEmbed permet de personnaliser l’apparence finale d’une timeline intégrée en définissant les propriétés correspondantes dans le balisage HTML, interprétées par le JavaScript de X inclus par défaut dans la réponse HTML. Le format du balisage renvoyé peut évoluer au fil du temps, à mesure que X ajoute de nouvelles fonctionnalités ou ajuste la représentation de sa timeline.

    Pour une timeline X spécifiée par l’URL de la timeline, au format JSON compatible [oEmbed](https://oembed.com/). Les timelines d’utilisateurs et de listes sont prises en charge. Le balisage de la timeline est destiné à être mis en cache sur vos serveurs pendant la durée de cache suggérée indiquée par la propriété cache_age.

    ## URL de la ressource

    *https://publish.x.com/oembed*

    ## Informations sur la ressource

    |     |     |
    | :--- | :--- |
    | Formats de réponse | JSON |
    | Authentification requise ? | Non  |
    | Limitation du débit | Non  |

    ## Paramètres

    | Nom | Description | Exemple |
    | :--- | :--- | :--- |
    | **url** | L’URL de la timeline X à intégrer | *   https://x.com/TwitterDev<br /> *https://x.com/TwitterDev/lists/national-parks
    | limit | Afficher jusqu’à N éléments, où N est une valeur comprise entre 1 et 20 inclus | 6   |
    | maxwidth | Définir la largeur maximale du widget. Doit être comprise entre 180 et 1200 inclus | 300 |
    | maxheight | Définir la hauteur maximale du widget. Doit être supérieure à 200 | 400 |
    | omit_script | Ne pas inclure d’élément script dans la réponse | 1   |
    | lang | Un [code de langue](/fr/en/docs/twitter-for-websites/twitter-for-websites-supported-languages/overview "Twitter language code") X pris en charge | es  |
    | theme | Lorsque défini sur dark, la timeline s’affiche avec un texte clair sur un arrière-plan sombre | dark |
    | chrome | Supprimer un composant d’affichage de la timeline avec des jetons séparés par des espaces<br /><br />*   noheader - masque l’en-tête<br />*   nofooter - masque le pied de page, s’il est visible<br />*   noborders - supprime toutes les bordures : autour du widget, entre les Tweets et à l’intérieur d’un Tweet<br />*   noscrollbar - rogne et masque la barre de défilement de la timeline, si visible<br />*   transparent - supprime la couleur d’arrière-plan | noheader%20nofooter |
    | aria_polite | Définir une valeur [ARIA live region politeness](https://www.w3.org/TR/wai-aria/states_and_properties#aria-live) assertive pour les Tweets ajoutés à une timeline | assertive |
    | dnt | Lorsqu’il est défini sur true, la timeline et sa page intégrée sur votre site ne sont pas utilisées à des fins incluant les [suggestions personnalisées](https://support.x.com/articles/20169421) et les [publicités personnalisées](https://support.x.com/articles/20170405) | true |

    ## Exemples de requêtes

    ```bash
    curl --request GET --url 'https://publish.x.com/oembed?url=https%3A%2F%2Ftwitter.com%2FInterior%2Fstatus%2F507185938620219395'
    twurl -H publish.x.com "/oembed?url=https://x.com/Interior/status/463440424141459456"
    ```

    ## Exemple de réponse

    ```json

    {
      "url": "https://x.com/TwitterDev",
      "title": "",
      "html": "<a class=\"twitter-timeline\" href=\"https://x.com/TwitterDev\">Tweets by TwitterDev</a>\n<script async src=\"//platform.x.com/widgets.js\" charset=\"utf-8\"></script>",
      "width": null,
      "height": null,
      "type": "rich",
      "cache_age": "3153600000",
      "provider_name": "Twitter",
      "provider_url": "https://x.com",
      "version": "1.0"
    }
    ```

  <Tab title="Embedded Tweets">
    L’extrait HTML renvoyé sera automatiquement reconnu comme un [Tweet intégré](https://developer.x.com/web/embedded-tweets) lorsque [le JavaScript du widget X est inclus dans la page](https://developer.x.com/web/javascript/loading).

    L’endpoint oEmbed permet de personnaliser l’apparence finale d’un Tweet intégré en définissant les propriétés correspondantes dans le balisage HTML, interprété par le JavaScript de X inclus par défaut avec la réponse HTML. Le format du balisage renvoyé peut évoluer au fil du temps, à mesure que X ajoute de nouvelles fonctionnalités ou ajuste la représentation d’un Tweet.

    Le balisage de secours d’un Tweet est destiné à être mis en cache sur vos serveurs pendant une durée maximale correspondant à la durée de cache suggérée, indiquée par la propriété `cache_age`.

    ## URL de ressource

    *https://publish.x.com/oembed*

    ## Informations sur les ressources

    |     |     |
    | :--- | :--- |
    | Formats de réponse | JSON |
    | Authentification requise ? | Non  |
    | Limitation de débit | Non  |

    ## Paramètres

    | Nom | Valeur par défaut | Description |
    | :--- | :--- | :--- |
    | `url` obligatoire  <br />String |     | L’URL du Tweet à intégrer |
    | `maxwidth`  <br />Int `[220..550]` | `325` | Largeur maximale d’un Tweet affiché, en pixels entiers. Toute valeur fournie en dehors de la plage autorisée sera ajustée à la largeur minimale ou maximale prise en charge, respectivement ; la valeur de largeur ainsi réinitialisée sera indiquée dans la propriété `width` renvoyée. Notez que X ne prend pas en charge le paramètre oEmbed `maxheight`. Les Tweets sont essentiellement du texte et ont donc une hauteur imprévisible qui ne peut pas être mise à l’échelle comme une image ou une vidéo. En conséquence, la réponse oEmbed ne fournira pas de valeur pour `height`. Les implémentations qui nécessitent des hauteurs uniformes pour les Tweets devraient se référer aux paramètres `hide_thread` et `hide_media` ci-dessous. |
    | `hide_media`  <br />Boolean, String ou Int | `false` | Lorsqu’il est défini sur `true`, `"t"` ou `1`, les liens dans un Tweet ne sont pas développés en aperçus de photos, de vidéos ou de liens. |
    | `hide_thread`  <br />Boolean, String ou Int | `false` | Lorsqu’il est défini sur `true`, `"t"` ou `1`, une version réduite du Tweet précédent dans un fil de conversation ne sera pas affichée lorsque le Tweet demandé est une réponse à un autre Tweet. |
    | `omit_script`  <br />Boolean, String ou Int | `false` | Lorsqu’il est défini sur `true`, `"t"` ou `1`, le `<script>` chargé de récupérer `widgets.js` ne sera pas renvoyé. Vos pages web doivent inclure leur propre référence à `widgets.js` pour l’utiliser avec tous les widgets X, y compris les [Tweets intégrés](https://developer.x.com/web/embedded-tweets). |
    | `align`  <br />Enum `{left,right,center,none}` | `none` | Indique si le Tweet intégré doit être aligné à gauche, à droite ou au centre dans la page par rapport à l’élément parent. |
    | `lang`  <br />Enum([Language](https://developer.x.com/en/docs/twitter-for-websites/twitter-for-websites-supported-languages/overview)) | `en` | Demande un HTML de réponse et un Tweet rendu dans la [langue X prise en charge par les Tweets intégrés](https://developer.x.com/web/overview/languages) spécifiée. |
    | `theme`  <br />Enum `{light, dark}` | `light` | Lorsqu’il est défini sur `dark`, le Tweet est affiché avec un texte clair sur fond sombre. |
    | `dnt`  <br />Boolean | `false` | Lorsqu’il est défini sur `true`, le Tweet et sa page intégrée sur votre site ne sont pas utilisés à des fins incluant les [suggestions personnalisées](https://support.x.com/articles/20169421) et les [publicités personnalisées](https://support.x.com/articles/20170405). |

    ## Exemples de requêtes

    ```bash
    curl --request GET --url 'https://publish.x.com/oembed?url=https%3A%2F%2Ftwitter.com%2Ftwiterdev'
    twurl -H publish.x.com "/oembed?url=https://x.com/TwitterDev"
    ```
