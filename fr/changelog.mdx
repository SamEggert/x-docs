<Update label="Aug 3, 2022" description="X API v2">
  ### Ajout d’alias d’opérateurs de filtrage

  Des alias d’opérateurs de filtrage/requête ont été ajoutés aux endpoints search et filtered stream, sur enterprise, premium et v2. Dans de nombreux cas, l’opérateur fait référence aux posts plutôt qu’aux statuses, par ex. « in&#95;reply&#95;to&#95;tweet&#95;id » comme alias de « in&#95;reply&#95;to&#95;status&#95;id ». Dans d’autres cas, le nouvel alias d’opérateur apporte plus de clarté au filtrage effectué. Par exemple, puisque les posts ne peuvent avoir qu’une seule pièce jointe vidéo, un nouvel alias « has:video&#95;link » est proposé pour « has:videos ». Consultez le [tableau des opérateurs v2 filtered stream](https://developer.x.com/en/docs/twitter-api/tweets/filtered-stream/integrate/build-a-rule#list) pour d’autres exemples.

<Update label="Dec 14, 2021" description="X API v2: Authentication">
  ### Prise en charge de l’authentification OAuth 2.0

  Désormais, tous les développeurs peuvent s’authentifier avec OAuth 2.0 en sélectionnant OAuth 2.0 comme méthode d’authentification dans le portail développeur. Nous avons ajouté la prise en charge des clients confidentiels et publics ainsi que de tous les endpoints v2 pertinents pour utiliser cette méthode d’authentification dans le cadre de cette version.

  OAuth 2.0 est un protocole d’autorisation standard du secteur qui offre aux développeurs un meilleur contrôle sur les portées (scopes) d’une application et améliore les flux d’autorisation sur plusieurs appareils. En d’autres termes, les développeurs créant des applications pour les personnes sur X auront davantage de contrôle sur les informations que leur app demande à leurs utilisateurs, afin que vous n’ayez à demander à vos utilisateurs finaux que les données et informations nécessaires.

  Ce protocole d’autorisation moderne vous permettra de proposer à vos utilisateurs finaux un parcours de consentement plus fluide pour autoriser votre app, en n’affichant que les portées spécifiques que vous avez demandées.

  Pour en savoir plus sur ce lancement, veuillez consulter notre annonce sur le forum.
</Update>

<Update label="Nov 15, 2021" description="X API v2: Early Access">
  ### Mises à jour majeures de la plateforme vers X API v2

  **Mises à jour majeures de la plateforme : X API v2 est désormais la version principale, nouveaux niveaux d’accès Essential et Elevated, et modifications de la politique**

  Aujourd’hui, nous annonçons que X API v2 est désormais la version principale de l’API X. Nous avons lancé suffisamment de endpoints et de fonctionnalités dans X API v2 pour répondre aux besoins de 90 % des applications existantes bâties sur l’API X.

  Nous lançons aujourd’hui les changements suivants pour améliorer encore l’expérience développeur avec X API v2 :

  * Accès Essential rapide et gratuit à l’API, et accès Elevated gratuit pour les développeurs dont les cas d’usage ont été approuvés. Vous trouverez davantage de détails sur ces niveaux d’accès sur notre page About the X API.

  * Nous supprimons, dans notre Developer Policy, des formulations qui restreignaient la façon de tirer parti des fonctionnalités essentielles de X et limitaient le nombre d’utilisateurs que votre application peut prendre en charge.

    * Nous ouvrons notre plateforme pour vous encourager à créer des outils et des produits qui rendent X meilleur, plus sain, et qui enrichissent la conversation publique. [Voir une liste de solutions que nous aimerions vous voir créer](https://developer.x.com/en/docs/twitter-api/what-to-build).

    * Plus précisément, nous avons supprimé des termes qui restreignaient la reproduction de l’expérience X, y compris ses fonctionnalités essentielles, ainsi que des clauses exigeant une autorisation pour disposer d’un grand nombre de jetons utilisateur.

    * Nous savons que créer des solutions qui aident les personnes sur X implique souvent qu’un développeur doive créer (ou reproduire) certains éléments disponibles sur X. Ces changements à notre Developer Policy visent à apporter de la clarté à l’écosystème des développeurs et à fournir une plateforme API ouverte qui facilite la création, l’innovation et l’impact sur la conversation publique.

  Nous détaillons bien davantage ce lancement, ainsi que ceux de X API qui ont conduit à ce moment, dans notre [publication sur le forum](https://devcommunity.x.com/t/ushering-in-a-new-era-for-the-twitter-developer-platform-with-the-twitter-api-v2/162087).

  En plus d’examiner les entrées antérieures du changelog, nous avons rassemblé des ressources clés pour comprendre tout ce qui a été publié dans X API v2 à ce jour :

  * [X API endpoint map](/fr/x-api/migrate/x-api-endpoint-map)

  * [About the X API](/fr/x-api/getting-started/about-x-api)
</Update>

<Update label="May 10, 2018" description="X API: Standard v1.1">
  ### Modifications du rendu du payload de citation

  Aujourd’hui, nous modifions la façon dont les URL sont rendues dans le payload de citation.

  Nous ajoutons une nouvelle entité appelée &quot;quoted&#95;status&#95;permalink&quot; afin de garantir que la publication citée puisse être référencée, et nous supprimerons le lien t.co du champ &quot;text&quot; de la publication citée.

  Avec le nouveau format, le lien t.co n’est plus ajouté au champ &quot;text&quot; de la publication citée dans l’objet &quot;quoted&#95;status&quot;.

  En outre, nous n’ajouterons plus l’URL t.co, l’expanded&#95;url et la display&#95;url de la publication citée dans l’entité &quot;urls&quot;. À la place, elles seront ajoutées à un nouvel objet &quot;quoted&#95;status&#95;permalink&quot; avec les attributs d’URL &quot;url&quot;, &quot;expanded&quot; et &quot;display&quot;. Cela signifie que l’objet &quot;quoted&#95;status&#95;permalink&quot; sera réservé au lien t.co renvoyant vers la publication citée, tandis que le tableau &quot;entities.urls&quot; pourra servir à identifier les liens partagés dans la publication d’origine ou dans la citation.

  Veuillez consulter notre [publication sur le forum](https://devcommunity.x.com/t/updating-how-urls-are-rendered-in-the-quote-tweet-payload/105473) et la [documentation mise à jour](https://devcommunity.x.com/t/upcoming-changes-to-the-developer-platform/104603) pour plus de détails.
