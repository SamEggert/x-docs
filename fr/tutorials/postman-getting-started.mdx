<div id="introduction">
  ## Introduction
</div>

Postman est une application de bureau et une application web qui vous permet d’effectuer des requêtes vers une API depuis une interface graphique. Nous recommandons d’utiliser Postman avec X API, X Ads API et les endpoints Labs pour explorer les fonctionnalités des API, ainsi que pour résoudre les problèmes de votre application.

Nous proposons actuellement deux collections Postman :

<div id="prerequisites">
  ### Prérequis
</div>

Avant de commencer à utiliser les collections Postman de X, assurez-vous de disposer des droits d’accès et des identifiants adaptés pour l’outil de la plateforme développeur X que vous souhaitez utiliser. Pour en savoir plus sur l’obtention de l’accès, consultez notre [page « Getting started »](/fr/resources/platform-overview).

Une fois que vous avez un [compte développeur](https://developer.x.com/en/portal/petition/essential/basic-info), que vous avez configuré une [application développeur](/fr/resources/fundamentals/developer-apps), que vous disposez d’un jeu de clés et de jetons d’[authentification](/fr/resources/fundamentals/authentication), et que vous avez correctement configuré votre environnement pour envoyer des requêtes à l’API spécifique que vous prévoyez d’utiliser, vous pouvez suivre les étapes ci-dessous pour commencer.
 

<div id="getting-started-with-xs-postman-collections">
  ## Premiers pas avec les collections Postman de X
</div>

<div id="step-one-add-one-of-the-x-postman-collections-to-your-account">
  ### Première étape : ajoutez l’une des collections Postman de X à votre compte
</div>

Vous pourriez configurer manuellement les points de terminaison que vous souhaitez utiliser dans Postman, mais nous avons déjà fait le plus gros du travail en préparant une collection prête à l’emploi des API pertinentes. Cliquez simplement sur l’un des liens de la section précédente « Collections Postman » ; une collection contenant tous les points de terminaison associés à l’API sélectionnée sera ajoutée à votre application Postman. Ces collections sont également disponibles dans le [réseau d’API Postman](https://explore.postman.com/). Chaque point de terminaison inclut automatiquement les paramètres disponibles, des exemples de réponses et le type d’authentification préconfiguré ; vous n’avez donc qu’à ajouter vos identifiants et les valeurs de paramètres pour commencer à explorer les fonctionnalités.

Dans cet exemple, nous allons utiliser la [collection API v2](https://app.getpostman.com/run-collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400) de X. 
 

<div id="step-two-add-your-keys-and-tokens-as-environmental-variables">
  ### Étape deux : ajoutez vos clés et jetons en tant que variables d’environnement
</div>

Lorsque vous ajoutez la collection à votre instance Postman, un environnement nommé « X API v2 » est automatiquement créé, dans lequel vous devrez ajouter vos clés et jetons. Dans cette étape, nous vous guidons pour ajouter vos clés et jetons depuis votre application développeur dans l’environnement « X API v2 ». 

Pour ajouter vos clés et jetons à l’environnement « X API v2 », cliquez sur le bouton « manage environments » dans le coin supérieur droit de Postman. Vous pouvez également cliquer sur le bouton des paramètres dans le coin supérieur droit. 
 

<Frame>
  <img src="/images/using-postman-1.png.twimg.1920.png" alt="Cette image montre le bouton « manage environments » mis en évidence dans la console Postman." />
</Frame>

Dans la liste des environnements, cliquez sur « X API v2 ». 

Ensuite, ajoutez une variable pour chacune des clés et chacun des jetons que vous avez générés via le tableau de bord Apps, dans le tableau. Voici un exemple de ce à quoi votre tableau devrait ressembler :

| VARIABLE | VALEUR INITIALE | VALEUR ACTUELLE |
| :--- | :--- | :--- |
| consumer&#95;key | `QAktM6W6DF6F7XXXXXX` | `QAktM6W6DF6F7XXXXXX` |
| consumer&#95;secret | `AJX560A2Omgwyjr6Mml2esedujnZLHXXXXXX` | `AJX560A2Omgwyjr6Mml2esedujnZLHXXXXXX` |
| access&#95;token | `1995XXXXX-0NGqVhk3s96IX6SgT3H2bbjOPjcyQXXXXXXX` | `1995XXXXX-0NGqVhk3s96IX6SgT3H2bbjOPjcyQXXXXXXX` |
| token&#95;secret | `rHVuh7dgDuJCOGeoe4tndtjKwWiDjBZHLaZXXXXXX` | `rHVuh7dgDuJCOGeoe4tndtjKwWiDjBZHLaZXXXXXX` |
| bearer&#95;token | `AAAAAAAAAAAAAAAAAAAAAL9v6AAAAAAA99t03huuqRYg0mpYAAFRbPR3XXXXXXX` | `AAAAAAAAAAAAAAAAAAAAAL9v6AAAAAAA99t03huuqRYg0mpYAAFRbPR3XXXXXXX` |

Notez que les clés et jetons utilisés dans le tableau ci-dessus sont fictifs et ne fonctionneront pas s’ils sont utilisés dans une requête. 

Une fois que vous avez ajouté vos identifiants comme variables et vérifié que l’environnement « X API v2 » est sélectionné, vous êtes prêt à effectuer des requêtes vers la collection X API v2. En effet, chaque onglet d’autorisation d’un endpoint est configuré pour hériter automatiquement les variables de cet environnement. 

Pour utiliser Postman avec des jetons d’accès utilisateur (User Access Tokens), passez à XXXX pour consulter les détails supplémentaires.

<div id="step-three-select-an-endpoint">
  ### Étape trois : sélectionner un endpoint
</div>

L’étape suivante consiste à choisir un endpoint dans la collection et à commencer à construire votre requête. Vous pouvez sélectionner un endpoint à partir de la navigation située à droite. Voici à quoi cela ressemble :

<Frame>
  <img src="/images/using-postman-2.png.twimg.1920.png" alt="Cette image montre la requête « Single Post » sélectionnée sous le menu déroulant « Post Lookup » dans la section « X API v2 »." />
</Frame>

Pour cet exemple, nous allons utiliser l’endpoint X API v2 &gt; Post Lookup &gt; Single Post. 

<div id="step-four-add-values-to-the-params-tab">
  #### Quatrième étape : ajouter des valeurs dans l’onglet Params
</div>

L’étape suivante consiste à accéder à l’onglet Params. Vous devriez voir un ensemble de paramètres inactifs avec des descriptions indiquant leur fonction, ainsi qu’une liste de toutes les valeurs possibles que vous pouvez transmettre avec votre requête.

Dans cet exemple, nous allons activer les paramètres de requête expansions et tweet.fields et ajouter les valeurs suivantes :

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `tweet.fields` | `created_at,attachments` |
| expansions | author&#95;id |

En plus d’ajouter les paramètres de requête, nous devons ajouter la variable de chemin requise, id. Étant donné que cet endpoint renvoie des Posts, nous devons fournir un ID de Post valide comme valeur.

Vous pouvez trouver l’ID du Post en vous rendant sur x.com, en cliquant sur un Post, puis en regardant l’URL. Par exemple, l’ID du Post de l’URL suivante est `1228393702244134912` :

`https://x.com/XDevelopers/status/1228393702244134912`

Dans l’onglet Params, faites défiler au-delà de tous les paramètres de requête pour afficher la section « Path Variables ». Nous allons ajouter l’ID du Post que vous souhaitez utiliser comme valeur pour la clé id.

Si vous avez tout saisi correctement à cette étape, l’onglet Params devrait ressembler à ce qui suit :

<div id="step-five-send-your-request-and-review-your-response">
  #### Cinquième étape : envoyez votre requête et vérifiez votre réponse
</div>

Maintenant que tout est en place dans votre requête, vous pouvez cliquer sur le bouton « Send ».

Si tout a été correctement configuré, vous devriez recevoir la charge utile suivante :

```json
{
    "data": {
        "author_id": "2244994945",
        "text": "What did the developer write in their Valentine’s card?\n  \nwhile(true) {\n    I = Love(You);  \n}",
        "id": "1228393702244134912",
        "created_at": "2020-02-14T19:00:55.000Z"
    },
    "includes": {
        "users": [
            {
                "username": "XDevelopers",
                "name": "Developers",
                "id": "2244994945"
            }
        ]
    }
}
```

  ### Générer un jeton d’accès utilisateur avec Postman :
</div>

<div id="using-oauth-10a-to-generate-a-user-access-token">
  #### Utiliser OAuth 1.0a pour générer un jeton d’accès utilisateur
</div>

Consultez le processus en trois étapes utilisé dans la [collection de test du flux OAuth 1.0a](https://www.postman.com/xapidevelopers/workspace/twitter-s-public-workspace/collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400?ctx=documentation).

<div id="using-oauth-20-to-generate-a-user-access-token">
  #### Utilisation d’OAuth 2.0 pour générer un jeton d’accès utilisateur
</div>

Si vous souhaitez générer un jeton d’accès utilisateur OAuth 2.0 dans Postman, vous pouvez créer des jetons d’accès OAuth 2.0 à utiliser avec la [collection Postman de l’API v2](https://www.postman.com/xapidevelopers/workspace/twitter-s-public-workspace/collection/9956214-784efcda-ed4c-4491-a4c0-a26470a67400?ctx=documentation) de X.

Cliquez sur la collection dans votre espace de travail, allez à l’onglet « Auth » et sélectionnez le type « OAuth 2.0 ». Ensuite, sous l’en-tête « Configure New Token », rendez-vous dans « Configuration Options ». Vous pouvez définir « Grant Type » sur « Authorization Code (With PKCE) ».

Mettez à jour votre URL de rappel pour qu’elle corresponde à l’URL de rappel associée à votre application. Mettez également à jour les paramètres suivants :

* Auth URL -  https://x.com/i/oauth2/authorize
* Access Token URL -  https://api.x.com/2/oauth2/token
* Client ID - Votre identifiant client OAuth 2.0 depuis le Dev Portal
* Client Secret - Si vous utilisez un client confidentiel
* Scope - Périmètres correspondant aux endpoints auxquels vous souhaitez accéder. Exemple : « tweet.read users.read »
* Votre URL de rappel (également appelée URL de redirection). Elle doit correspondre à celle définie dans les paramètres d’authentification de votre application.
* State - state

Quand vous êtes prêt, cliquez sur « Get New Access Token » pour générer un jeton d’accès. Si une boîte de dialogue indique qu’une erreur s’est produite, vous devrez peut-être appuyer sur le bouton Retour pour vous connecter. Vous devrez autoriser votre application à accéder à votre compte en cliquant sur « Authorize app » dans la boîte de dialogue.

Après avoir autorisé votre application, vous serez redirigé vers Postman, où vous pourrez voir votre jeton et cliquer sur le bouton « Use Token » pour commencer à envoyer des requêtes au nom d’un utilisateur autorisé.

Vous êtes maintenant prêt à utiliser la collection Postman.

<div id="whats-next">
  ## Et ensuite
</div>

Si vous cliquez sur le bouton « Code » dans Postman, vous pouvez convertir la requête que nous venons de créer dans le langage de votre choix, comme Python, Node ou Ruby, pour vous aider à démarrer. Postman propose une [excellente documentation](https://learning.getpostman.com/) qui pourra vous être utile. Nous avons également du [code d’exemple sur GitHub](https://github.com/xdevplatform) pour vous aider à vous intégrer plus rapidement aux points de terminaison.
