<div id="how-to-get-access-to-the-ads-api">
  ## Comment accéder à l’API Ads
</div>

1. Créez un [compte développeur](https://developer.x.com/en/apply-for-access).
2. Créez une [application développeur](https://developer.x.com/app) et sécurisez votre jeton.
3. Rendez-vous sur [ads.x.com/help](https://ads.x.com/help) pour soumettre une demande d’accès à l’API Ads pour chacune de vos applications développeur.

<Note>**Remarque :** Si vous travaillez déjà sur la X Developer Platform et disposez d’un compte développeur, passez directement à l’étape trois.</Note>

<div id="step-one-signup-for-a-developer-account">
  ### Première étape : Inscription à un compte développeur
</div>

Pour envoyer une requête à n’importe quel produit d’API de X, vous devez d’abord créer un compte développeur. Dans le portail développeur, créez un projet et une application développeur. Cela vous fournira un jeu d’identifiants que vous utiliserez pour authentifier toutes les requêtes auprès de l’API.

<Button href="https://developer.x.com/en/apply-for-access">
  Créer un compte développeur
</Button>

<div id="step-two-save-your-apps-key-and-tokens-and-keep-them-secure">
  ### Deuxième étape : Enregistrez la clé et les jetons de votre app et gardez-les en sécurité
</div>

Dans votre app développeur, un ensemble de clés API (également appelées Consumer Keys) vous est fourni. Vous aurez également la possibilité de générer un ensemble de jetons d’accès pouvant être utilisés pour effectuer des requêtes au nom de votre compte X personnel, ainsi qu’un jeton Bearer permettant d’authentifier les endpoints nécessitant OAuth 2.0 Bearer Token. Comme ces clés et jetons n’expirent pas sauf s’ils sont régénérés, nous vous suggérons de créer des variables d’environnement ou d’utiliser un gestionnaire de mots de passe sécurisé.

Veuillez également noter votre App ID, qui se trouve dans l’URL des détails de votre app. Cela facilitera la demande d’accès à l’Ads API à l’étape suivante.

<Note>
**Remarque :** Vos clés et jetons ne s’afficheront qu’une seule fois dans le portail développeur, il est donc important d’enregistrer ces informations d’identification dans votre gestionnaire de mots de passe dès leur génération.

Si vous égarez ou oubliez vos clés et jetons, vous devrez les régénérer, ce qui crée de nouvelles clés et de nouveaux jetons et invalide les anciens. Vous devrez alors mettre à jour toute intégration que vous auriez configurée avec vos identifiants précédents.

En savoir plus sur nos [meilleures pratiques en matière d’authentification](https://developer.x.com/en/docs/authentication/guides/authentication-best-practices).
</Note>

<div id="step-three-apply-for-access-to-the-ads-api">
  ### Troisième étape : demander l’accès à l’API Ads
</div>

À ce stade, vous disposez [d’un accès de base à l’API X](https://developer.x.com/en/docs/twitter-api/v1), mais pas aux fonctionnalités spécifiques de l’API X Ads. Vous devez maintenant demander l’accès et obtenir l’approbation pour l’API Ads. Rendez-vous sur [ads.x.com/help](https://ads.x.com/help) pour soumettre une demande d’accès à l’API Ads pour chacune de vos applications développeur.

<div id="access-tiers">
  ### Niveaux d’accès
</div>

Dans le cadre du processus de demande, vous devrez préciser le niveau d’accès dont vous avez besoin. En savoir plus sur les [autorisations au niveau de l’application et du compte publicitaire](https://developer.x.com/en/docs/twitter-ads-api/obtaining-ads-account-access#levels-of-access).

<div id="conversion-only">
  #### Conversion Only
</div>

Accès aux points de terminaison de conversion mobile et web avec droits de lecture et d’écriture.

<div id="standard-access">
  #### Accès standard
</div>

Accès aux endpoints Analytics, Campaign Management, Creatives, Custom Audiences et Conversion avec droits de lecture et d’écriture.

<Note>**Remarque :** Une fois votre application approuvée pour l’accès à l’Ads API, vous devrez [régénérer les jetons d’accès utilisateur](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens) afin d’effectuer correctement des [requêtes authentifiées vers l’Ads API](https://developer.x.com/en/docs/twitter-ads-api/making-authenticated-requests).</Note>

<Note>**Remarque :** Les développeurs de l’Ads API ayant demandé l’accès avant juillet 2023 peuvent disposer de niveaux d’accès et d’autorisations différents et être limités à cinq jetons OAuth. Consultez notre guide sur [l’augmentation de l’accès](https://developer.x.com/en/docs/twitter-ads-api/increasing-access) pour obtenir l’accès à des endpoints supplémentaires ou lever les limites de jetons pour les applications existantes.</Note>

<div id="make-your-first-request">
  ## Effectuer votre première requête
</div>

Pour tester votre accès à l’Ads API, envoyez une requête à l’endpoint [GET accounts](https://developer.x.com/en/docs/twitter-ads-api/campaign-management/api-reference/accounts). Cet endpoint renvoie les comptes publicitaires auxquels l’utilisateur actuellement autorisé a accès. Utilisez les ID de comptes publicitaires issus de cette requête dans vos requêtes API suivantes pour lire et écrire des données pour un compte publicitaire donné. En utilisant [Twurl](https://developer.x.com/en/docs/tutorials/using-twurl) en ligne de commande, la requête serait la suivante.

**Exemple de requête**

```bash
twurl -H ads-api.x.com "/11/accounts"
```

**Exemple de réponse**

