---
title: Pagination
---

Afin de prendre en charge un nombre maximal plus élevé de campagnes et de permettre la récupération efficace de toutes les entités associées à un compte, l’API Ads prend désormais en charge la pagination sur de nombreux endpoints GET. Le mécanisme de pagination est simple à utiliser et très proche de la pagination à base de curseurs de l’API REST, comme décrit dans [Utilisation de curseurs pour parcourir des collections](https://developer.x.com/en/docs/x-api/v1/pagination).

<div id="getting-started">
  ## Pour commencer
</div>

Pour certaines requêtes GET, nous acceptons désormais les paramètres de requête `cursor` et `count`, tous deux facultatifs. Si une réponse contient plus de `count` entités, ces endpoints renverront les `count` premières entités ainsi qu’une clé `next_cursor` dans le JSON de réponse.

`GET https://ads-api.x.com/5/accounts/abc1/campaigns?count=50`

```json
{
  "data": \[...\],
  "next_cursor": "c-3yvu1pzhd3i7",
  "request": {...}
}
```

Pour récupérer la série suivante de résultats, ajoutez `cursor` à vos paramètres de requête :

`GET https://ads-api.x.com/5/accounts/abc1/campaigns?cursor=c-3yvu1pzhd3i7&count=50`

```json
{
  "data": \[...\],
  "next_cursor": "c-3w3zdyg8ywan",
  "request": {...}
}
```

<div id="going-deeper">
  ## Approfondir
</div>

Pour la plupart des endpoints, la valeur maximale de `count` est **1 000**, la minimale est **1**, et la valeur par défaut est **200**.

La valeur fournie par `next_cursor` est toujours une chaîne et doit être considérée comme opaque ; l’implémentation est susceptible d’évoluer. Si moins de `count` entités sont renvoyées dans la page actuelle de l’ensemble de résultats, la valeur de `next_cursor` sera `null`.

**Remarque :** les endpoints d’analytics ne prennent pas en charge ce mode de pagination. La pagination sur les endpoints de statistiques est gérée en spécifiant des périodes temporelles. Consultez la documentation de ces endpoints pour plus d’informations.
