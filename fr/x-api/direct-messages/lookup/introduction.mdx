Cette première version de la recherche de Messages privés inclut trois méthodes GET :

* **GET /2/dm_conversations/with/:participant_id/dm_events** - Récupère les événements de Messages privés associés à une conversation en tête-à-tête. Le paramètre de chemin :participant_id correspond à l’ID utilisateur du compte qui converse avec l’utilisateur authentifié effectuant cette requête.
* **GET /2/dm_conversations/:dm_conversation_id/dm_events** - Récupère les événements de Messages privés associés à un ID de conversation spécifique, tel qu’indiqué par le paramètre de chemin :dm_conversation_id.
* **GET /2/dm_events** - Récupère les événements de Messages privés associés à un utilisateur, y compris les conversations en tête-à-tête et les conversations de groupe. Des événements datant d’au plus 30 jours sont disponibles.

Notez que les ID d’événements de Messages privés sont communs entre les versions v1.1 et v2 (ainsi que dans l’application X), de sorte que la méthode v1.1 permettant de lister un événement unique peut être utilisée avec ces nouveaux endpoints v2. Notez également que les Enterprise et Premium Account Activity APIs prennent en charge les messages en tête-à-tête v2, mais ne prennent pas encore en charge les conversations de groupe.

Avec cette version, trois types d’événements sont pris en charge, et ces endpoints acceptent des paramètres de requête pour filtrer selon ceux-ci :

* **MessageCreate** - Un message a été créé.
* **ParticipantsJoin** - Un nouveau participant a rejoint une conversation.
* **ParticipantsLeave** - Un participant a quitté une conversation.

Une limite de 300 requêtes par utilisateur sur 15 minutes s’applique aux méthodes GET. Cette limite est partagée entre ces endpoints GET.

Comme vous effectuez des requêtes pour le compte d’un utilisateur avec les endpoints de Messages privés v2, vous devez vous authentifier soit avec [OAuth 1.0a User Context](/fr/resources/fundamentals/authentication), soit avec [OAuth 2.0 Authorization Code with PKCE](/fr/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3), en utilisant des jetons d’accès associés aux utilisateurs ayant autorisé votre application X. Pour générer ces jetons d’accès avec OAuth 1.0a, vous pouvez utiliser le [flux OAuth à 3 acteurs](/fr/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow). Pour générer des jetons d’accès utilisateur avec OAuth 2.0, vous pouvez utiliser le [flux Authorization Code with PKCE](/fr/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2).

**Configuration du compte**

Pour accéder à ces endpoints, vous aurez besoin de :

* D’un [compte développeur](https://developer.x.com/en/portal/petition/essential/basic-info) approuvé.
* De vous authentifier en utilisant les clés et jetons d’une [application développeur](/fr/resources/fundamentals/developer-apps) située dans un [projet](/fr/resources/fundamentals/projects).

En savoir plus sur l’accès aux endpoints de l’API X v2 dans notre [guide de démarrage](/fr/x-api/getting-started/getting-access).
