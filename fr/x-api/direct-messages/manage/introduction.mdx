Les Messages privés permettent des conversations privées sur X. Les Messages privés sont l’une des fonctionnalités les plus populaires de X, avec un large éventail de cas d’usage. Ces cas d’usage vont des discussions de groupe entre amis à la gestion du service client pour des marques du monde entier. De nouvelles versions v2 des endpoints des Messages privés seront introduites par étapes, et cette première étape inclut des endpoints essentiels pour créer des Messages privés et répertorier les événements de conversation en Messages privés. Pour la première fois, l’API X v2 prend en charge les conversations de groupe.

Cette version initiale de Manage Direct Messages inclut trois méthodes POST pour créer des Messages privés :

* **POST /2/dm&#95;conversations/with/:participant&#95;id/messages** - Crée un Message privé en tête‑à‑tête. Cette méthode crée soit une nouvelle conversation 1‑1, soit récupère la conversation en cours et y ajoute le Message privé. Le paramètre de chemin :participant&#95;id est l’ID utilisateur du compte qui reçoit le message.
* **POST /2/dm&#95;conversations** - Crée une nouvelle conversation de groupe et y ajoute un Message privé. Ces requêtes nécessitent une liste de participants à la conversation. Notez que vous pouvez créer plusieurs conversations avec la même liste de participants. Ces requêtes renverront toujours un nouvel ID de conversation.
* **POST /2/dm&#95;conversations/:dm&#95;conversation&#95;id/messages** - Crée un Message privé et l’ajoute à une conversation existante. Le paramètre de chemin :dm&#95;conversation&#95;id est l’ID de la conversation à laquelle le message sera ajouté.

Notez que les ID d’événements de Message privé sont communs entre la v1.1 et la v2 (ainsi que l’application X), de sorte que les méthodes v1.1 pour masquer/supprimer des Messages privés peuvent être utilisées avec ce nouvel endpoint v2. Notez également que les Enterprise et Premium Account Activity APIs prennent en charge les messages 1‑1 v2, mais ne prennent pas encore en charge les conversations de groupe.

Il existe une limite de débit utilisateur de 200 requêtes par 15 minutes pour la méthode POST. Il existe également une limite de 1 000 requêtes par 24 heures et par utilisateur. De plus, il existe une limite de 15 000 requêtes par 24 heures. Notez que ces limites de débit sont partagées entre ces endpoints POST.

Puisque vous effectuez des requêtes au nom d’un utilisateur avec les endpoints Manage Posts, vous devez vous authentifier soit avec [OAuth 1.0a User Context](https://developer-staging.x.com/resources/fundamentals/authentication), soit avec [OAuth 2.0 Authorization Code with PKCE](fr/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2), et utiliser un jeton d’accès utilisateur associé à un utilisateur ayant autorisé votre App. Pour générer ce jeton d’accès utilisateur avec OAuth 1.0a, vous pouvez utiliser le [flux OAuth à 3 volets](/fr/resources/fundamentals/authentication#using-and-generating-an-app-only-bearer-token). Pour générer un jeton d’accès utilisateur avec OAuth 2.0, vous pouvez utiliser le [flux Authorization Code with PKCE](/fr/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)).

<Note>
  **Configuration du compte**

  Pour accéder à ces endpoints, vous aurez besoin de :

  * D’un [compte développeur](https://developer.x.com/en/portal/petition/essential/basic-info) approuvé.
  * De vous authentifier à l’aide des clés et des tokens d’une [developer App](/fr/resources/fundamentals/developer-apps) située dans un [Project](/fr/resources/fundamentals/projects).

  En savoir plus sur l’accès aux endpoints de l’API X v2 dans notre [guide de démarrage](/fr/x-api/getting-started/getting-access).
</Note>
