  ## Comparaison des endpoints de recherche d’utilisateurs de l’API X
</div>

Les endpoints de recherche d’utilisateurs v2 remplaceront les endpoints standard v1.1 [GET users/lookup](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-users-lookup.html) et [GET users/show](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/get-users-show.html). Si vous avez du code, des applications ou des outils qui utilisent l’une de ces versions des endpoints de recherche d’utilisateurs et que vous envisagez de migrer vers le nouvel endpoint X API v2, cette série de guides est faite pour vous. 
 

Les tableaux suivants comparent les différents types d’endpoints de recherche d’utilisateurs :
 

|     |     |     |
| :--- | :--- | :--- |
| **Description** | **Standard v1.1** | **X API v2** |
| Méthodes HTTP prises en charge | `GET` | `GET` |
| Domaine hôte | `https://api.x.com` | `https://api.x.com` |
| Chemin de l’endpoint | `/1.1/users/show.json` `/1.1/users/lookup.json` | `/2/users`<br /><br />`/2/users/:id`<br /><br />`/2/users/by`<br /><br />`/2/users/by/:username` |
| [Authentification](/fr/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context<br /><br />App only<br /><br />OAuth 2.0 Authorization Code with PKCE |
| [Limites de débit](/fr/resources/fundamentals/rate-limits) des requêtes par défaut | 900 requêtes par 15 min (par utilisateur)<br /><br />/show - 900 requêtes par 15 min (par application)  <br />/lookup - 300 requêtes par 15 min (par application) | 900 requêtes par 15 min (par utilisateur)<br /><br />300 requêtes par 15 min (par application) |
| Nombre maximal d’utilisateurs par réponse | /show -  1<br /><br />/lookup - 100 | 100 |
| Format de l’objet de réponse JSON | Format standard v1.1 | [Format X API v2](/fr/x-api/fundamentals/data-dictionary) (déterminé par les paramètres de requête fields et expansions, non rétrocompatible avec les formats v1.1)<br /><br />Pour en savoir plus sur la migration du format standard v1.1 vers le format X API v2, veuillez consulter notre [guide de migration des formats de données](/fr/x-api/migrate/data-format-migration). |
| Prend en charge la sélection des [fields](/fr/x-api/fundamentals/data-dictionary) à retourner dans la charge utile |     | ✔   |
| Prend en charge les champs [annotations](/fr/x-api/fundamentals/post-annotations) (sur la publication épinglée) |     | ✔   |
| Prend en charge la demande de nouveaux champs [metrics](/fr/x-api/fundamentals/metrics) (sur la publication épinglée) |     | ✔   |
| Prend en charge le champ [conversation_id](/fr/x-api/fundamentals/conversation-id) (sur la publication épinglée) |     | ✔   |
| Nécessite l’utilisation d’identifiants provenant d’une [developer App](/fr/resources/fundamentals/developer-apps) associée à un [project](/fr/resources/fundamentals/projects) |     | ✔   |
