<div id="blocks-lookup">
  ### Recherche des blocages
</div>

L’endpoint GET de recherche des blocages vous permet de voir quels comptes vous avez bloqués pour le compte d’un utilisateur autorisé. Cet endpoint est limité à 15 requêtes par tranche de 15 minutes et par utilisateur.

Comme vous effectuez des requêtes d’informations privées avec la recherche des blocages et agissez au nom d’un utilisateur disposant des autorisations de gestion des blocages, vous devez authentifier ces endpoints avec [OAuth 1.0a (contexte utilisateur)](/fr/resources/fundamentals/authentication) ou [OAuth 2.0 (Authorization Code avec PKCE)](fr/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2), et utiliser des jetons d’accès utilisateur associés à un utilisateur ayant autorisé votre application. Ces jetons peuvent être générés via le [flux OAuth à 3 étapes](/fr/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) ou le [flux d’autorisation Authorization Code avec PKCE](/fr/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0).

<Note>
  **Configuration du compte**

  Pour accéder à ces endpoints, vous aurez besoin de :

  * D’un [compte développeur](https://developer.x.com/en/portal/petition/essential/basic-info) approuvé.
  * De vous authentifier à l’aide des clés et jetons d’une [application développeur](/fr/resources/fundamentals/developer-apps) qui se trouve dans un [projet](/fr/resources/fundamentals/projects).

  En savoir plus sur l’accès aux endpoints de l’API X v2 dans notre [guide de démarrage](/fr/x-api/getting-started/getting-access).
</Note>
