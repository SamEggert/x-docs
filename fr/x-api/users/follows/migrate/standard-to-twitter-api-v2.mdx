<div id="manage-follows-standard-v11-compared-to-x-api-v2">
  #### Gérer les abonnements : Standard v1.1 et X API v2
</div>

Si vous avez travaillé avec les points de terminaison standard v1.1 [POST friendships/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/post-friendships-create) et [POST friendships/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/follow-search-get-users/api-reference/post-friendships-destroy), ce guide a pour objectif de vous aider à comprendre les similitudes et les différences entre les points de terminaison de gestion des abonnements du Standard et de X API v2.

* **Similitudes**
  * Contexte utilisateur OAuth 1.0a
* **Différences**
  * URL des points de terminaison
  * Exigences pour l’application et le projet
  * Méthodes HTTP
  * Paramètres de requête

     

<div id="similarities">
  #### Similitudes
</div>

**Méthode d’authentification OAuth 1.0a en contexte utilisateur**

Les deux versions de l’endpoint prennent en charge [OAuth 1.0a User Context](/fr/resources/fundamentals/authentication#oauth-1-0a-2). Par conséquent, si vous utilisiez auparavant l’un des endpoints standard v1.1 pour gérer les suivis, vous pouvez continuer à utiliser la même méthode d’authentification en migrant vers la version X API v2. 

<div id="differences">
  #### Différences
</div>

**URL des endpoints**

* Endpoints Standard v1.1 :
  * POST https://api.x.com/1.1/friendships/create.json
    (suivre un utilisateur)
  * POST https://api.x.com/1.1/friendships/destroy.json
    (se désabonner d’un utilisateur)
* Endpoint X API v2 :
  * POST https://api.x.com/2/users/:id/following
    (suivre un utilisateur)
  * DELETE https://api.x.com/2/users/:source&#95;user&#95;id/following/:target&#95;user&#95;id
    (se désabonner d’un utilisateur) 

**Exigences relatives à l’App et au Project**

Les endpoints X API v2 exigent que vous utilisiez des identifiants provenant d’une [developer App](/fr/resources/fundamentals/developer-apps) associée à un [Project](/fr/resources/fundamentals/projects) lors de l’authentification de vos requêtes. Tous les endpoints X API v1.1 peuvent utiliser des identifiants d’Apps autonomes ou d’Apps associées à un Project.

**Paramètres de requête**

Les paramètres de requête Standard v1.1 suivants ont des équivalents dans X API v2 :

| Standard v1.1 | X API v2 |
| :--- | :--- |
| Aucun équivalent | id (POST), source&#95;user&#95;id (DELETE) |
| user&#95;id | target&#95;user&#95;id |
| screen&#95;name | Aucun équivalent |

Veuillez noter que les paramètres Standard v1.1 sont transmis en tant que paramètres de requête, tandis que les paramètres X API v2 sont transmis en tant que paramètres de corps (pour l’endpoint POST) ou de chemin (pour l’endpoint DELETE).

De plus, les paramètres v2 id et source&#95;user&#95;id ne sont pas requis lors de l’utilisation des endpoints Standard v1.1, puisque les Access Tokens transmis avec OAuth 1.0a User Context permettaient de déduire quel utilisateur initiait l’action suivre/se désabonner.
