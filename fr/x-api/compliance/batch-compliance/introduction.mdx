X s’engage auprès de notre communauté de développeurs qui utilisent l’API X. Dans le cadre de cet engagement, nous visons à rendre notre API ouverte et équitable pour les développeurs, sûre pour les personnes sur X et bénéfique pour la plateforme X dans son ensemble. Il est crucial que tout développeur qui stocke du contenu X hors ligne s’assure que les données reflètent l’intention de l’utilisateur et l’état actuel du contenu sur X. Par exemple, lorsque quelqu’un sur X supprime une publication ou son compte, protège ses publications, ou supprime les informations de géolocalisation de ses publications, il est essentiel, tant pour X que pour nos développeurs, de respecter les attentes et l’intention de cette personne. Les endpoints de conformité par lot offrent aux développeurs un outil simple pour aider à maintenir les données X en conformité avec le [X Developer Agreement and Policy](https://developer.x.com/en/developer-terms/policy). 

Ces endpoints de conformité par lot vous permettent de téléverser de grands ensembles d’ID de publications ou d’utilisateurs afin de récupérer leur statut de conformité et de déterminer quelles données nécessitent une action pour mettre vos jeux de données en conformité. Veuillez noter que l’utilisation des endpoints de conformité par lot est limitée aux cas d’usage susmentionnés, et que toute autre finalité est interdite et peut entraîner des mesures d’application.

En général, 4 étapes sont nécessaires pour utiliser cet endpoint :

1. **Créer un job de conformité**
   Vous pouvez spécifier le type de job (avec la valeur tweets ou users) pour indiquer si le jeu de données que vous souhaitez téléverser contient des ID de publications ou des ID d’utilisateurs. Vous pouvez avoir un job simultané par type de job à tout moment.
2. **Téléverser votre jeu de données vers l’upload&#95;url**
   Ensuite, vous téléversez votre jeu de données sous forme de fichier texte brut vers l’upload&#95;url fournie, chaque ligne du fichier contenant un seul ID de publication ou un seul ID d’utilisateur. L’upload&#95;url expire après 15 minutes.
3. **(Facultatif) Vérifier l’état du job**
   Vous pouvez vérifier l’état de votre job de conformité pour voir s’il est created, in&#95;progress, failed ou complete.
4. **Télécharger les résultats**
   Lorsque votre job est terminé, vous pouvez télécharger les résultats en utilisant le download&#95;url. Le download&#95;url expire après une semaine (à partir de la création du job).

   Le résultat contiendra un ensemble d’objets JSON (un objet par ligne). Chaque objet contiendra un ID de publication, la date de création de la publication (utile pour retrouver des publications par date), l’action requise, la raison de l’action de conformité et la date à laquelle l’utilisateur a été suspendu.

Vous recevrez les types d’événements de conformité suivants dans vos résultats :

* deleted - indique que la publication ou le compte utilisateur a été supprimé
* deactivated - indique que la publication ou le compte utilisateur a été désactivé
* scrub&#95;geo - indique que les informations de géolocalisation associées à la publication ou à l’utilisateur ont été supprimées
* protected - indique que le compte à l’origine de la publication est passé en privé
* suspended - indique que le compte à l’origine de la publication a été suspendu

**Configuration du compte**

Pour accéder à ces endpoints, vous aurez besoin de :

* D’un [developer account](https://developer.x.com/en/portal/petition/essential/basic-info) approuvé.
* De vous authentifier en utilisant les clés et jetons provenant d’une [developer App](/fr/resources/fundamentals/developer-apps) située au sein d’un [Project](/fr/resources/fundamentals/projects). 

Pour en savoir plus sur l’accès aux endpoints de l’API X v2, consultez notre [getting started guide](/fr/x-api/getting-started/getting-access).
