X s’engage auprès de notre communauté de développeurs qui créent avec l’API X. Dans le cadre de cet engagement, nous visons à rendre notre API ouverte et équitable pour les développeurs, sûre pour les personnes sur X et bénéfique pour la plateforme X dans son ensemble. Il est crucial que tout développeur qui stocke du contenu X hors ligne s’assure que les données reflètent l’intention de l’utilisateur et l’état actuel du contenu sur X. Par exemple, lorsqu’une personne sur X supprime une publication ou son compte, protège ses publications ou modifie une publication, il est essentiel, pour X comme pour nos développeurs, de respecter les attentes et l’intention de cette personne.

Des flux en temps réel d’événements de conformité fournissent aux développeurs les outils pour maintenir les données X en conformité avec la [X Developer Agreement and Policy](https://developer.x.com/en/developer-terms/policy).

Il existe deux flux d’événements de conformité, l’un pour les événements de conformité des publications, et l’autre pour les événements de conformité des utilisateurs. Ces flux sont disponibles avec un accès Enterprise et sont conçus pour aider les partenaires qui ingèrent de gros volumes de données à « écouter » les événements de conformité tels que les événements d’édition de publication.

Ces flux fournissent les événements suivants :

**Flux de conformité des publications :**

* delete - indique que la publication a été supprimée.

* tweet&#95;edit - indique qu’une publication a été modifiée et fournit l’ID de la publication mise à jour.

* withheld - indique que la publication a été restreinte dans un ou plusieurs pays.

* drop - indique que la publication doit être retirée de l’affichage public.

* undrop - indique que la publication peut à nouveau être affichée et traitée comme publique.

**Flux de conformité des utilisateurs :**

* **user&#95;delete** - indique que le compte utilisateur a été supprimé

* **user&#95;undelete** - indique que le compte utilisateur a été restauré

* **user&#95;protect** - indique que le compte utilisateur est passé en privé

* **user&#95;unprotect** - indique que le compte utilisateur est repassé en public

* **user&#95;withheld** - indique que le compte utilisateur a été restreint dans un ou plusieurs pays.

* **user&#95;suspend** - indique que le compte utilisateur a été suspendu

* **user&#95;unsuspend** - indique que le compte utilisateur a été rétabli

* **user&#95;profile&#95;modification** - indique que le profil utilisateur a été mis à jour. Cela inclut la description, le nom, la localisation et l’URL mis à jour.

**Configuration du compte**

Pour accéder à ces endpoints, vous aurez besoin de :

* D’un [compte développeur](https://developer.x.com/en/portal/petition/essential/basic-info) approuvé.
* De vous authentifier à l’aide des clés et jetons d’une [developer App](/fr/resources/fundamentals/developer-apps) située dans un [Project](/fr/resources/fundamentals/projects).

En savoir plus sur l’accès aux endpoints de X API v2 dans notre [guide de démarrage](/fr/x-api/getting-started/getting-access).
