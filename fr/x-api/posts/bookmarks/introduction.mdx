<div id="manage-bookmarks">
  ### Gérer les signets
</div>

Nous proposons deux méthodes pour gérer les signets : POST et DELETE. La méthode POST permet de créer des signets. De même, la méthode DELETE permet de supprimer un signet spécifique.

Une limite par utilisateur de 50 requêtes par période de 15 minutes s’applique aux méthodes POST et DELETE.

Comme vous effectuez des requêtes pour le compte d’un utilisateur avec les endpoints de gestion des signets, vous devez vous authentifier en générant un jeton d’accès utilisateur avec OAuth 2.0. Vous pouvez utiliser le [flux d’autorisation Code avec PKCE](/fr/resources/fundamentals/authentication/oauth-2-0/user-access-token) à cette fin. Pour utiliser cet endpoint, vous devez inclure les scopes `tweet.read`, `users.read` et `bookmark.write`.

<div id="bookmarks-lookup">
  ### Recherche de signets
</div>

L’endpoint de recherche de signets propose une seule méthode, GET. Cette méthode vous permet de récupérer des signets depuis votre propre compte ou un compte authentifié. Des jetons de pagination sont fournis pour parcourir de grands ensembles de résultats avec cet endpoint.

Une limite de 180 requêtes par utilisateur s’applique sur une fenêtre de 15 minutes pour la méthode GET. Avec cet endpoint, vous pourrez récupérer jusqu’à 800 de vos posts récemment ajoutés aux signets.

Comme vous effectuez des requêtes pour le compte d’un utilisateur avec les endpoints de recherche de signets, vous devez vous authentifier en générant un jeton d’accès utilisateur avec OAuth 2.0. Vous pouvez utiliser le [flux Autorisation par code avec PKCE](/fr/resources/fundamentals/authentication/oauth-2-0/user-access-token) à cet effet. Pour utiliser cet endpoint, vous devez inclure les scopes `tweet.read`, `users.read` et `bookmark.read`.

**Configuration du compte**

Pour accéder à ces endpoints, vous aurez besoin de :

* D’un [compte développeur](https://developer.x.com/en/portal/petition/essential/basic-info) approuvé.
* De vous authentifier à l’aide des clés et jetons d’une [application développeur](/fr/resources/fundamentals/developer-apps) située dans un [projet](/fr/resources/fundamentals/projects).

Pour en savoir plus sur l’accès aux endpoints de l’API X v2, consultez notre [guide de démarrage](/fr/x-api/getting-started/getting-access).
