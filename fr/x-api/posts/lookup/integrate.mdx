<div id="authentication">
  ### Authentification
</div>

Tous les points de terminaison de l’API X v2 nécessitent des requêtes authentifiées. Vous pouvez vous authentifier avec :

* [Contexte utilisateur OAuth 1.0a](/fr/resources/fundamentals/authentication) en utilisant des clés d’API, des jetons d’accès et des paramètres supplémentaires pour [créer un en-tête d’autorisation](/fr/resources/fundamentals/authentication#authorizing-a-request).
* [OAuth 2.0 App-Only](/fr/resources/fundamentals/authentication#oauth-2-0) en joignant un [jeton d’accès d’application](/fr/resources/fundamentals/authentication) à votre requête.
* [OAuth 2.0 Authorization Code avec PKCE](/fr/resources/fundamentals/authentication) pour un meilleur contrôle de la portée de l’application et l’autorisation multi-appareils.

<Note>
  OAuth 1.0a peut être difficile à implémenter. Si vous n’êtes pas familier avec ce protocole, envisagez d’utiliser une [bibliothèque](/fr/resources/tools-and-libraries) ou OAuth 2.0 pour vos requêtes. Pour les métriques privées ou les Posts, utilisez le contexte utilisateur OAuth 1.0a ou OAuth 2.0 Authorization Code avec PKCE.
</Note>

<Tip>
  **Veuillez noter**

  > Si vous demandez les champs suivants, le contexte utilisateur OAuth 1.0a ou OAuth 2.0 Authorization Code est requis :
  >
  > * `tweet.fields.non_public_metrics`
  > * `tweet.fields.promoted_metrics`
  > * `tweet.fields.organic_metrics`
  > * `media.fields.non_public_metrics`
  > * `media.fields.promoted_metrics`
  > * `media.fields.organic_metrics`
</Tip>

<div id="developer-portal-projects-and-apps">
  ### Portail développeur, Projets et Apps
</div>

Pour obtenir des identifiants pour X API v2, vous devez disposer de :

1. Un [compte développeur](/fr/resources/fundamentals/developer-portal) approuvé.
2. D’un [projet](/fr/resources/fundamentals/projects) au sein de ce compte développeur.
3. D’une [app développeur](/fr/resources/fundamentals/developer-apps) dans ce projet, où se trouvent les clés et les jetons.

<div id="rate-limits">
  ### Limites de taux
</div>

Les requêtes à l’API X sont soumises à des [limites de taux](/fr/resources/fundamentals/rate-limits) afin de gérer le volume. Les limites s’appliquent à la fois au niveau de l’application et au niveau de l’utilisateur :

* **Niveau application** : limite le nombre de requêtes effectuées par période par toute application.
* **Niveau utilisateur** : limite la fréquence à laquelle un utilisateur authentifié peut effectuer des recherches de posts sur l’ensemble des applications développeur.

<div id="fields-and-expansions">
  ### Champs et expansions
</div>

L’API X v2 permet de sélectionner des champs de données spécifiques à l’aide de `fields` et `expansions` :

* **Expansions** : Permettent de récupérer des objets liés supplémentaires. Les expansions prises en charge incluent :
  * `edit_history_tweet_ids`
  * `attachments.poll_ids`
  * `attachments.media_keys`
  * `author_id`
  * `entities.mentions.username`
  * `geo.place_id`
  * `in_reply_to_user_id`
  * `referenced_tweets.id`
  * `referenced_tweets.id.author_id`

* **Fields** : Indiquent les champs de données au sein des objets pour renvoyer des informations supplémentaires. Par défaut, l’objet Post renvoie `id`, `text` et `edit_history_tweet_ids`. D’autres options, comme `tweet.created_at` et `tweet.entities`, doivent être demandées explicitement.

Pour en savoir plus, consultez le guide sur les champs et les expansions dans le [X API v2 data dictionary](/fr/x-api/fundamentals/data-dictionary).

<div id="post-edits">
  ### Modifications de Posts
</div>

Les Posts éligibles peuvent être modifiés jusqu’à cinq fois dans les 30 minutes suivant leur publication. L’endpoint de consultation des Posts fournit toujours la dernière version du Post. Pour les cas d’usage quasi temps réel, tenez compte de cette fenêtre. Pour en savoir plus, consultez [Notions de base sur la modification des Posts](/fr/x-api/fundamentals/edit-posts).

<div id="edge-cases">
  ### Cas limites
</div>

* **Métriques sponsorisées** : La demande de métriques sponsorisées pour des Posts non sponsorisés renvoie une réponse vide.
* **Texte tronqué** : Le texte des Posts est tronqué pour les Retweets. Pour récupérer le texte complet, développez le Post référencé.
