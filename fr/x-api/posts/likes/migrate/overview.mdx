<div id="comparing-x-apis-likes-endpoints">
  ## Comparaison des endpoints Likes de l’API X
</div>

Ces guides porteront sur les points suivants :

* **Paramètres de requête de l’API** — L’endpoint de l’API X v2 introduit un nouvel ensemble de paramètres de requête. Certains vous seront familiers, notamment si vous avez intégré Labs, mais il existe de nombreuses différences importantes, comme l’introduction des paramètres [fields](/fr/x-api/fundamentals/fields) et [expansions](/fr/x-api/fundamentals/expansions).

* **Exigences relatives aux Apps et aux Projects** — Pour accéder à l’API X v2, vous devrez utiliser des identifiants provenant d’une [developer App](/fr/resources/fundamentals/developer-apps) associée à un [Project](/fr/resources/fundamentals/projects)

<div id="likes-lookup">
  #### Consultation des J'aime
</div>

**Utilisateurs ayant aimé une publication**

Le point de terminaison des utilisateurs ayant aimé est une nouvelle fonctionnalité de la v2, qui vous permet d&#39;obtenir des informations sur les utilisateurs ayant aimé une publication.

| Description | X API v2 |
| :--- | :--- |
| Méthodes HTTP prises en charge | GET |
| Domaine hôte | https://api.x.com |
| Chemin du point de terminaison | /2/tweets/:id/liking_users |
| [Authentification](/fr/resources/fundamentals/authentication) | Jeton d&#39;accès OAuth 2.0 (Bearer)<br /><br />Contexte utilisateur OAuth 1.0a |
| [Limites de débit](/fr/resources/fundamentals/rate-limits) de requête par défaut | 75 requêtes par 15 min (par application)<br /><br />75 requêtes par 15 min (par utilisateur) |

**Publications aimées par un utilisateur**

Les tableaux suivants comparent le point de terminaison standard v1.1 [GET favorites/list](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/get-favorites-list) et les points de terminaison X API v2 pour les publications aimées :

| Description | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| Méthodes HTTP prises en charge | GET | GET |
| Domaine hôte | https://api.x.com | https://api.x.com |
| Chemin du point de terminaison | /1.1/favorites/list.json | /2/users/:id/liked_tweets |
| [Authentification](/fr/resources/fundamentals/authentication) | Contexte utilisateur OAuth 1.0a | Jeton d&#39;accès OAuth 2.0 (Bearer)<br /><br />Contexte utilisateur OAuth 1.0a |
| [Limites de débit](/fr/resources/fundamentals/rate-limits) de requête par défaut | 75 requêtes par 15 min | 75 requêtes par 15 min (par application)<br /><br />75 requêtes par 15 min (par utilisateur) |
| Formats de données | Format Standard v1.1 | [Format X API v2](/fr/x-api/fundamentals/data-dictionary) (déterminé par les paramètres de requête fields et expansions, non rétrocompatible avec les formats v1.1)<br /><br />Pour en savoir plus sur la migration du format Standard v1.1 vers le format X API v2, veuillez consulter notre [guide de migration des formats de données](/fr/x-api/migrate/data-format-migration). |

<div id="manage-likes">
  #### Gérer les J’aime
</div>

Les endpoints v2 de gestion des J’aime remplacent les endpoints v1.1 [POST favorites/create](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-create) et [POST favorites/destroy](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-favorites-destroy).

Les tableaux suivants comparent les endpoints standard de la v1.1 et de l’API X v2 pour gérer les J’aime :

<div id="like-a-post">
  #### Aimer une publication
</div>

| Description | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| Méthodes HTTP prises en charge | POST | POST |
| Domaine hôte | https://api.x.com | https://api.x.com |
| Chemin d’endpoint | /1.1/favorites/create.json | /2/users/:id/likes |
| [Authentification](/fr/resources/fundamentals/authentication) | Contexte utilisateur OAuth 1.0a | Contexte utilisateur OAuth 1.0a |
| [Limites de débit](/fr/resources/fundamentals/rate-limits) par défaut pour les requêtes | 1000 requêtes par 24 heures (par utilisateur)<br /><br />1000 requêtes par 24 heures (par application) | 50 requêtes par 15 min (par utilisateur)<br /><br />1000 requêtes par 24 heures (par utilisateur, partagé avec DELETE) |

<div id="unlike-a-post">
  #### Retirer un J’aime d’une publication
</div>

| Description | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| Méthodes HTTP prises en charge | POST | DELETE |
| Domaine hôte | https://api.x.com | https://api.x.com |
| Chemin d’endpoint | /1.1/favorites/destroy.json | /2/users/:id/likes/:tweet_id |
| [Authentification](/fr/resources/fundamentals/authentication) | Contexte utilisateur OAuth 1.0a | Contexte utilisateur OAuth 1.0a |
| [Limites de débit](/fr/resources/fundamentals/rate-limits) par défaut des requêtes | 1000 requêtes par 24 heures (par utilisateur)<br /><br />1000 requêtes par 24 heures (par application) | 50 requêtes par 15 min (par utilisateur)<br /><br />1000 requêtes par 24 heures (par utilisateur, partagées avec POST) |
