<div id="comparing-x-apis-manage-posts-endpoints">
  ## Comparaison des endpoints de gestion des Posts de l’API X
</div>

Les endpoints v2 de gestion des Posts remplaceront les endpoints standard v1.1 [POST statuses/update](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-update) et [POST statuses/destroy/:id](https://developer.x.com/en/docs/twitter-api/v1/tweets/post-and-engage/api-reference/post-statuses-destroy-id). Si vous avez du code, des applications ou des outils qui utilisent la version v1.1 des endpoints de gestion des Posts et envisagez de migrer vers le nouvel endpoint de l’API X v2, cet ensemble de guides est fait pour vous.

Les tableaux suivants comparent les endpoints standard v1.1 et les endpoints v2 de gestion des Posts de l’API X :

<div id="create-a-post">
  ### Créer une publication
</div>

| Description | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| Méthodes HTTP prises en charge | POST | POST |
| Domaine hôte | https://api.x.com | https://api.x.com |
| Chemin d’endpoint | /1.1/statuses/update.json | /2/tweets |
| [Authentification](/fr/resources/fundamentals/authentication) | Contexte utilisateur OAuth 1.0a | Contexte utilisateur OAuth 1.0a<br /><br />Code d’autorisation OAuth 2.0 avec PKCE |
| [Limites de débit](/fr/x-api/fundamentals/rate-limits) par défaut des requêtes | Aucune<br /><br />300 requêtes par fenêtre de 3 heures, par utilisateur et par application. Partagé avec l’endpoint POST Retweets v1.1. | 200 requêtes par tranche de 15 minutes, par utilisateur<br /><br />300 requêtes par fenêtre de 3 heures, par utilisateur et par application. Partagé avec l’endpoint POST Retweets v2. |
| Nécessite l’utilisation d’identifiants provenant d’une [application développeur](/fr/resources/fundamentals/developer-apps) associée à un [projet](/fr/resources/fundamentals/projects) |     | ✔   |

<div id="delete-a-post">
  ### Supprimer une publication
</div>

| Description | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| Méthodes HTTP prises en charge | POST | DELETE |
| Domaine hôte | https://api.x.com | https://api.x.com |
| Chemin d’endpoint | /1.1/statuses/destroy/:id.json | /2/tweets/:id |
| [Authentification](/fr/resources/fundamentals/authentication) | OAuth 1.0a Contexte utilisateur | OAuth 1.0a Contexte utilisateur<br /><br />OAuth 2.0 Code d’autorisation avec PKCE |
| [Limites de débit](/fr/x-api/fundamentals/rate-limits) par défaut des requêtes | Aucune | 50 requêtes par tranche de 15 min et par utilisateur |
| Requiert l’utilisation des identifiants d’une [application développeur](/fr/resources/fundamentals/developer-apps) associée à un [projet](/fr/resources/fundamentals/projects) |     | ✔   |
