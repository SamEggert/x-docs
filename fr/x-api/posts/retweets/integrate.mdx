<div id="authentication">
  #### Authentification
</div>

Tous les endpoints de X API v2 exigent que vous authentifiiez vos requêtes avec un ensemble d’identifiants, également appelés clés et jetons.

Vous pouvez utiliser soit OAuth 1.0a User Context, soit OAuth 2.0 Bearer Token pour authentifier vos requêtes vers l’endpoint de recherche de Retweets.

Les endpoints de gestion des Retweets nécessitent l’utilisation d’OAuth 1.0a User Context, ce qui signifie que vous devez utiliser un ensemble de clés API et de jetons d’accès utilisateur pour effectuer une requête avec succès. Les jetons d’accès doivent être associés à l’utilisateur pour le compte duquel vous effectuez la requête. Si vous souhaitez générer un ensemble de jetons d’accès pour un autre utilisateur, celui-ci doit autoriser ou authentifier votre application en utilisant le flux OAuth à 3 étapes (3-legged OAuth).

Veuillez noter qu’OAuth 1.0a peut être complexe à utiliser. Si vous n’êtes pas familier avec cette méthode d’authentification, nous vous recommandons d’utiliser une [bibliothèque](/fr/x-api/tools-and-libraries/overview) pour authentifier correctement vos requêtes.

**Veuillez noter**

Si vous demandez les champs suivants, OAuth 1.0a User Context est requis :

* tweet.fields.non&#95;public&#95;metrics
* tweet.fields.promoted&#95;metrics
* tweet.fields.organic&#95;metrics
* media.fields.non&#95;public&#95;metrics
* media.fields.promoted&#95;metrics
* media.fields.organic&#95;metrics

<div id="developer-portal-projects-and-developer-apps">
  #### Portail développeur, Projets et Apps développeur
</div>

Pour obtenir un ensemble d’informations d’authentification compatible avec les endpoints de l’API X v2, vous devez [créer un compte développeur](https://developer.x.com/en/portal/petition/essential/basic-info), configurer un [Projet](/fr/resources/fundamentals/projects) dans ce compte, puis créer une [App développeur](/fr/resources/fundamentals/developer-apps) dans ce Projet. Vous pourrez ensuite retrouver vos clés et jetons dans votre App développeur. 
 

<div id="rate-limits">
  #### Limites de taux
</div>

Chaque jour, plusieurs milliers de développeurs envoient des requêtes à l’API X. Pour gérer l’important volume de ces requêtes, des [limites de taux](/fr/x-api/fundamentals/rate-limits) sont appliquées à chaque endpoint, ce qui restreint le nombre de requêtes que vous pouvez effectuer au nom de votre application ou d’un utilisateur authentifié.

Les endpoints de gestion des Retweets sont limités à 50 requêtes par tranche de 15 minutes (par utilisateur). De plus, pour l’endpoint POST, vous êtes limité à 300 requêtes sur une fenêtre de 3 heures (par utilisateur et par application).

Avec l’endpoint de recherche de Retweets, vous êtes limité à 75 requêtes par tranche de 15 minutes. En outre, seuls les 100 utilisateurs ayant retweeté le plus récemment seront renvoyés par cet endpoint.

<div id="fields-and-expansions">
  #### Champs et expansions
</div>

L’API X v2 permet aux utilisateurs de sélectionner exactement quelles données ils souhaitent récupérer depuis l’API à l’aide d’un ensemble d’outils appelés champs et expansions. Le paramètre expansion vous permet de développer les objets référencés dans le payload. Par exemple, ce point de terminaison permet de récupérer les [expansions](/fr/x-api/fundamentals/expansions) suivantes :

* attachments.poll&#95;ids
* attachments.media&#95;keys
* author&#95;id, entities.mentions.username
* geo.place&#95;id
* in&#95;reply&#95;to&#95;user&#95;id,
* referenced&#95;tweets.id,
* referenced&#95;tweets.id.author&#95;id

Le paramètre fields vous permet de sélectionner précisément quels [champs](/fr/x-api/fundamentals/fields), au sein des différents objets de données, vous souhaitez recevoir. Ces points de terminaison renvoient principalement des objets Post. Par défaut, l’objet Post renvoie les champs id et text. Pour recevoir des champs supplémentaires tels que tweet.created&#95;at ou tweet.entities, vous devez les demander explicitement à l’aide d’un paramètre fields. Parmi les champs importants à envisager pour votre intégration figurent nos données de sondage, les métriques, les annotations de Post et l’ID de conversation.

Nous avons ajouté à notre [dictionnaire de données de l’API X v2](/fr/x-api/fundamentals/data-dictionary) un guide expliquant comment [utiliser conjointement les champs et les expansions](/fr/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions).
