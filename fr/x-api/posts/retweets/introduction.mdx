<div id="retweets-lookup">
  ### Recherche des Retweets
</div>

Avec le point de terminaison Retweets lookup, vous pouvez obtenir la liste des comptes ayant retweeté une publication. Pour cet endpoint, des jetons de pagination sont fournis pour parcourir de grands volumes de résultats par lots de jusqu’à 100 utilisateurs.

Vous pouvez authentifier ces endpoints avec [OAuth 1.0a (contexte utilisateur)](/fr/resources/fundamentals/authentication), [OAuth 2.0 (application uniquement)](https://developer.x.com/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token), ou [OAuth 2.0 (code d’autorisation avec PKCE)](/fr/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-3). 

<div id="retweets-of-me">
  ### Republications de mes publications
</div>

La recherche « Republications de mes publications » vous permet de voir lesquelles de vos propres publications ont été partagées par d&#39;autres.

Elle offre aux utilisateurs une visibilité sur les publications qui sont partagées et qui trouvent un écho auprès de leur audience, ce qui peut faciliter des interactions plus ciblées et des ajustements de la stratégie de contenu.

Vous pouvez authentifier ces endpoints avec [OAuth 1.0a User Context](/fr/resources/fundamentals/authentication) ou [OAuth 2.0 Authorization Code with PKCE](/fr/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2).

<div id="manage-retweets">
  #### Gérer les Retweets
</div>

Les endpoints de gestion des Retweets vous permettent de retweeter ou d’annuler le retweet d’une publication spécifiée au nom d’un compte authentifié. Pour ce groupe d’endpoints, deux méthodes sont disponibles : POST et DELETE. La méthode POST vous permet de retweeter une publication, et la méthode DELETE vous permet d’annuler le retweet d’une publication donnée.

Comme vous effectuez des requêtes au nom d’un utilisateur, vous devez authentifier ces endpoints avec [OAuth 1.0a User Context](/fr/resources/fundamentals/authentication) ou [OAuth 2.0 Authorization Code with PKCE](fr/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2), et utiliser les jetons d’accès utilisateur associés à l’utilisateur pour lequel vous effectuez la requête. Vous pouvez générer ce jeton d’accès utilisateur à l’aide du [flux OAuth à 3 volets](/fr/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow) (OAuth 1.0a) ou du [flux d’autorisation Authorization Code with PKCE](/fr/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce) (OAuth 2.0). Vous pouvez retweeter une publication depuis votre compte ou depuis le compte d’un utilisateur authentifié. Pour ces deux endpoints, la limite par utilisateur est de 50 requêtes par tranche de 15 minutes et par endpoint.
