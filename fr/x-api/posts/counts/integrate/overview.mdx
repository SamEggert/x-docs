</div>

<div id="code-samples">

</div>

Vous souhaitez configurer ces endpoints avec du code dans votre langage de programmation préféré ? Nous proposons plusieurs exemples de code sur notre [page GitHub](https://github.com/xdevplatform/Twitter-API-v2-sample-code) que vous pouvez utiliser comme point de départ, notamment un [client Python](https://github.com/xdevplatform/search-tweets-python).

<div id="libraries">
  #### Bibliothèques
</div>

Profitez de l’une de nos nombreuses [bibliothèques communautaires tierces](/fr/x-api/tools-and-libraries/overview) pour vous lancer. Vous pouvez trouver une bibliothèque compatible avec les points de terminaison v2 en recherchant le tag de version approprié.

<div id="postman">
  #### Postman
</div>

Postman est un excellent outil pour tester ces endpoints. Chaque requête Postman inclut tous les paramètres de l’endpoint concerné afin de vous aider à comprendre rapidement ce qui est à votre disposition. Pour en savoir plus sur nos collections Postman, consultez notre page [Utiliser Postman](/fr/tutorials/postman-getting-started).
 

<div id="key-concepts">
  ### Principes clés
</div>

<div id="authentication">
  #### Authentification
</div>

Tous les endpoints de X API v2 exigent que les requêtes soient [authentifiées](/fr/resources/fundamentals/authentication) à l’aide d’un ensemble d’identifiants, également appelés clés et jetons. Cet endpoint requiert l’utilisation d’[OAuth 2.0 Bearer Token](/fr/resources/fundamentals/authentication#oauth-2-0), ce qui signifie que vous devez transmettre un [Bearer Token](/fr/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) pour effectuer une requête avec succès. Vous pouvez générer un Bearer Token directement depuis une application développeur ou en générer un via l’endpoint [POST oauth2/token](/fr/resources/fundamentals/authentication#post-oauth2-token).

<div id="developer-portal-projects-and-developer-apps">
  #### Portail développeur, Projets et applications développeur
</div>

Pour utiliser n’importe quel point de terminaison de l’API X v2, vous devez disposer d’un [compte développeur](/fr/resources/fundamentals/developer-portal), configurer un [projet](/fr/resources/fundamentals/projects) dans ce compte, puis créer une [application développeur](/fr/resources/fundamentals/developer-apps) dans ce projet. Les clés et jetons de cette application fonctionneront avec les points de terminaison de comptage des Posts récents. Si vous souhaitez utiliser le point de terminaison de comptage des Posts de l’archive complète, ou tirer parti des opérateurs avancés et d’une longueur de requête plus importante, vous devrez avoir été approuvé pour l’accès Enterprise.

Veuillez consulter notre section sur l’accès Enterprise pour en savoir plus.

<div id="rate-limits">
  #### Limites de débit
</div>

Chaque jour, des dizaines de milliers de développeurs envoient des requêtes à l’API X. Pour gérer ce volume, des [limites de débit](/fr/x-api/fundamentals/rate-limits) sont appliquées à chaque endpoint afin de limiter le nombre de requêtes qu’un développeur peut effectuer pour le compte d’une application ou d’un utilisateur authentifié.

Cet endpoint est soumis à une limite de débit au niveau de l’application, ce qui signifie que vous, en tant que développeur, ne pouvez effectuer qu’un certain nombre de requêtes vers cet endpoint sur une période donnée depuis une application donnée (selon les identifiants que vous utilisez). 

<div id="building-queries">
  #### Création de requêtes
</div>

La fonctionnalité centrale de ces endpoints est l’usage d’une requête unique pour filtrer les Posts et produire les décomptes qui vous sont renvoyés. Ces requêtes sont constituées d’opérateurs qui ciblent des attributs des Posts et des utilisateurs, tels que des mots-clés, des hashtags et des URL. Les opérateurs peuvent être combinés au sein de requêtes à l’aide de logique booléenne et de parenthèses pour affiner le comportement de correspondance.

Vous pouvez consulter notre guide sur [la création d’une requête](/fr/x-api/posts/counts/integrate/build-a-query) pour en savoir plus.

<div id="pagination">
  #### Pagination
</div>

Pour les décomptes de posts récents, aucun next&#95;token n’est renvoyé, ce qui signifie que, quelle que soit la granularité, vous obtiendrez le volume de posts des 7 derniers jours en un seul appel API.

Pour les décomptes de posts de l’archive complète, vous obtiendrez des données pour les 30 derniers jours. Pour des données antérieures à 30 jours, vous recevrez un next&#95;token que vous pourrez utiliser pour paginer et récupérer les données supplémentaires. 

<div id="building-queries-for-post-counts">
  ### Créer des requêtes pour les décomptes de Posts
</div>

Les points de terminaison des décomptes de Posts acceptent une requête unique via une requête GET et renvoient un ensemble de décomptes historiques de Posts correspondant à la requête. Les requêtes sont composées d’opérateurs utilisés pour faire correspondre divers attributs de Post.
