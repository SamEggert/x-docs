<div id="comparing-x-apis-post-counts-endpoints">
  ## Comparaison des endpoints de comptage de Posts de l’API X
</div>

L’endpoint de comptage de Posts v2 finira par remplacer l’[endpoint de comptage de l’Enterprise Search API](/fr/x-api/enterprise-gnip-2.0/fundamentals/search-api#counts-requests-post-count). Si vous avez du code, des applications ou des outils qui utilisent une ancienne version d’un endpoint de comptage de Posts et envisagez de migrer vers les endpoints plus récents de l’API X v2, ce guide est fait pour vous.

Cette page contient deux tableaux de comparaison :

* [Comptes de Posts récents](#recent)
* [Comptes de Posts sur l’intégralité des archives](#full-archive)
* [Comparaison des opérateurs de filtrage](#operator)

<div id="recent-post-counts-comparison">
  ### Comparaison des décomptes récents de Posts
</div>

La version Enterprise des endpoints de décompte de Posts vous permet d’obtenir des décomptes soit sur 30 jours, soit sur l’archive complète. Par conséquent, l’endpoint v2 des décomptes récents de Posts, qui couvre une période de 7 jours, ne remplace directement aucun des endpoints susmentionnés.

Cependant, pour faciliter les comparaisons, nous allons examiner en quoi l’endpoint v2 des décomptes récents de Posts se compare à l’endpoint Enterprise sur 30 jours.

Le tableau suivant compare les différents types d’endpoints de décompte récent de Posts :

| **Description** | **Enterprise** | **X API v2** |
| :--- | :--- | :--- |
| Domaine hôte | https://gnip-api.x.com | https://api.x.com |
| Chemin d’endpoint | /search/30day/accounts/:account&#95;name/:label/counts.json | /2/tweets/counts/recent |
| [Authentication](/fr/resources/fundamentals/authentication) | Basic authentication | OAuth 2.0 Bearer Token |
| Format d’horodatage | YYYYMMDDhhmm | YYYY-MM-DDTHH:mm:ssZ  <br />[ISO 8601 / RFC 3339](https://tools.ietf.org/html/rfc3339#section-5.6) |
| Renvoie les décomptes de Posts âgés d’au plus | 31 jours | 7 jours |
| Méthodes HTTP prises en charge | GET | GET |
| Limites de taux de requêtes par défaut | 20 requêtes par 1 s, agrégées entre les données de recherche et les requêtes de décompte  <br />La limite par minute varie selon le partenaire, comme spécifié dans votre contrat. | 180 requêtes par 15 min par utilisateur  <br />450 requêtes par 15 min par application |
| Prend en charge le filtrage via les [annotations](/fr/x-api/fundamentals/post-annotations) |     | ✔   |
| Prend en charge le filtrage via [conversation&#95;id](/fr/x-api/fundamentals/conversation-id) |     | ✔   |
| Nom de la clé JSON pour le tableau de données de Posts | results | data |
| Granularité temporelle | Jour, heure ou minute | Jour, heure ou minute |
| Fuseau horaire | UTC | UTC |
| Paramètres de requête pour sélectionner la période | fromDate  <br />toDate | start&#95;time  <br />end&#95;time |
| Paramètres de requête pour naviguer par ID de Post |     | since&#95;id  <br />until&#95;id |
| Nécessite l’utilisation d’identifiants provenant d’une [developer App](/fr/resources/fundamentals/developer-apps) associée à un [project](/fr/resources/fundamentals/projects) |     | ✔   |

<div id="filtering-operator-comparison">
  ### Comparaison des opérateurs de filtrage
</div>

Les deux versions (Enterprise et v2) de Post counts diffèrent par les opérateurs disponibles et présentent également des niveaux de disponibilité variables au sein de chaque version, comme expliqué ci-dessous.

Enterprise

* Il n’existe pas de sous-niveaux d’opérateurs Enterprise. Tous les opérateurs Enterprise sont disponibles pour tous les utilisateurs Enterprise.

X API v2

* **Core :** Ces opérateurs sont disponibles pour tout utilisateur v2.
* **Advanced :** Ces opérateurs sont uniquement disponibles pour les utilisateurs approuvés pour l’accès Academic Research.

Vous pouvez en savoir plus sur chacun de ces ensembles d’opérateurs dans leurs guides respectifs :

* [Enterprise operators](/fr/x-api/enterprise-gnip-2.0/fundamentals/search-api#building-search-queries)
* [X API v2 operators](/fr/x-api/posts/search/integrate/build-a-query)
