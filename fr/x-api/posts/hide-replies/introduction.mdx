Cet endpoint vous permet de masquer ou d’afficher de nouveau des réponses par programmation selon des critères que vous définissez. Comme dans l’expérience principale de X, les réponses seront masquées dans la conversation principale mais resteront visibles sur une page distincte. Vous pouvez utiliser l’endpoint pour créer des applications aidant les utilisateurs à masquer les réponses qu’ils ne jugent pas pertinentes.

L’endpoint de masquage des réponses utilise soit [OAuth 1.0a User Context](/fr/resources/fundamentals/authentication#oauth-1-0a-2), soit [OAuth 2.0 Authorization Code with PKCE](fr/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) pour l’authentification. En cas de succès, l’endpoint masque une seule réponse publiée dans une conversation de Post initiée par un utilisateur authentifié. Chaque conversation prend en charge le masquage de jusqu’à 725 Posts.

<Note>
  **Configuration du compte**

  Pour accéder à ces endpoints, vous aurez besoin de :

  * D’un [compte développeur](https://developer.x.com/en/portal/petition/essential/basic-info) approuvé.
  * De vous authentifier à l’aide des clés et jetons d’une [App développeur](/fr/resources/fundamentals/developer-apps) qui se trouve dans un [Project](/fr/resources/fundamentals/projects).

  Pour en savoir plus sur l’accès aux endpoints de X API v2, consultez notre [guide de démarrage](/fr/x-api/getting-started/getting-access).
</Note>
