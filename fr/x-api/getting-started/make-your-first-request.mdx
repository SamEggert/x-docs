Ce guide vous présente les étapes à suivre pour effectuer votre première requête. Il s’agit d’une excellente ressource une fois que vous avez créé un compte X.

Si vous souhaitez utiliser des exemples de code, des guides plus techniques ou un outil graphique comme Postman, consultez les guides suivants pour effectuer votre première requête :

* [Guide de démarrage rapide pour effectuer votre première requête vers l’endpoint post](/fr/x-api/posts/manage-tweets/quickstart)
* [Prise en main de Postman](/fr/tutorials/postman-getting-started) - idéal pour les débutants
* [Exemples de code pour l’API X v2](https://github.com/xdevplatform/Twitter-API-v2-sample-code)
* [Référence de l’API de recherche de posts](/fr/x-api/posts/lookup/introduction)

Ce guide suppose que vous avez récupéré votre [clé API et secret](https://developer.x.com/resources/fundamentals/authentication/api-key-and-secret), votre [jeton d’accès utilisateur et secret](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens), ainsi que votre [jeton d’accès d’application](/fr/resources/fundamentals/authentication#bearer-token-also-known-as-app-only), et que vous les avez stockés dans un emplacement sécurisé. Pour savoir comment procéder, suivez les étapes du guide [obtenir l’accès à l’API X](/fr/x-api/getting-started/getting-access).

<div id="step-1-identify-which-endpoint-you-would-like-to-use">
  ### Étape 1. Identifier le point de terminaison que vous souhaitez utiliser
</div>

L’API X vous permet d’exécuter, via du code, une variété d’actions que vous pourriez aussi réaliser sur le site web ou l’application mobile de X.

Nous proposons une liste complète des points de terminaison disponibles dans notre [Index de référence de l’API](/fr/x-api/introduction), mais nous vous recommandons, par souci de simplicité, de vous en tenir à l’un des suivants :

* [Gérer les publications &gt; Publier un Tweet](/fr/x-api/posts/manage-tweets/introduction)
* [Rechercher des publications](/fr/x-api/posts/search/introduction)
* [Recherche d’utilisateurs](/fr/x-api/users/lookup/introduction)

<div id="step-2-choose-a-tool-to-make-your-request">
  ### Étape 2. Choisissez un outil pour formuler votre requête
</div>

Si certaines requêtes sont simples, d&#39;autres peuvent être plus difficiles à mettre en place. C&#39;est pourquoi l&#39;excellente communauté de développeurs a créé des outils pour masquer une partie de cette complexité.

Voici quelques outils recommandés et des indications sur la façon de les utiliser :

<div id="postman">
  #### Postman
</div>

Postman est un outil visuel que vous pouvez utiliser pour envoyer des requêtes vers des points de terminaison REST. Nous avons créé d’excellentes ressources sur Postman pour vous aider à démarrer et à explorer les différents points de terminaison disponibles via l’API X.

Nous vous recommandons de lire notre tutoriel [&quot;Bien démarrer avec Postman&quot;](/fr/tutorials/postman-getting-started) pour apprendre à ajouter vos clés et jetons et à effectuer votre première requête.

Nous avons également produit un guide de démarrage rapide pour chacun de nos points de terminaison de l’API X v2, dont la plupart utilisent Postman. Vous pouvez trouver ces guides dans chaque section de point de terminaison, mais voici des liens vers quelques-uns :

* Démarrage rapide : publier un Post
* Démarrage rapide : rechercher des Posts
* Démarrage rapide : rechercher un utilisateur

Veuillez noter que vous ne pouvez pas envoyer de requêtes aux points de terminaison de streaming avec Postman. Veuillez consulter le guide de démarrage rapide du [flux filtré](/fr/x-api/posts/filtered-stream/quickstart) ou du [flux échantillonné à 1 %](/fr/x-api/posts/sample-stream) pour apprendre à travailler avec ces points de terminaison.

Si vous préférez un outil graphique plus simple, vous pouvez également envisager d’utiliser [Insomnia](https://insomnia.rest).

<div id="sample-code">
  #### Exemples de code
</div>

Si vous souhaitez utiliser un peu de code simple pour effectuer votre requête, nous avons rassemblé des exemples de code dans plusieurs langages pour chacun de nos endpoints X API v2.

Vous pouvez trouver les exemples de code dans notre dépôt GitHub, [X-API-v2-sample-code](https://github.com/xdevplatform/Twitter-API-v2-sample-code), qui contient également un fichier README expliquant comment configurer vos identifiants afin d’utiliser correctement les requêtes.

Par exemple, voici un exemple cURL pour l’endpoint de recherche d’utilisateur. Pour utiliser cette requête, remplacez simplement $ACCESS_TOKEN et $USERNAME par votre [jeton d’accès d’application](/fr/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) et votre identifiant X. Ensuite, copiez-collez ce code dans votre outil en ligne de commande et appuyez sur « Return » ou « Enter ».

```bash
curl "https://api.x.com/2/users/by/username/$USERNAME" -H "Authorization: Bearer $ACCESS_TOKEN"
```

<div id="libraries">
  #### Bibliothèques
</div>

L’extraordinaire communauté de développeurs X a également créé des bibliothèques dans divers langages de programmation, utilisables pour envoyer des requêtes à l’API X.

Nous avons mis en place une page [&quot;Outils et bibliothèques&quot;](/fr/x-api/tools-and-libraries/overview) qui répertorie toutes les bibliothèques communautaires dont nous avons connaissance. Chaque bibliothèque devrait inclure un fichier README expliquant comment configurer le dépôt sur votre machine et effectuer votre première requête.

> **Veuillez noter :** Si vous rencontrez des problèmes, consultez la documentation pour l’endpoint auquel vous adressez une requête, notre [section d’assistance](https://developer.x.com/en/support), ou contactez la communauté sur nos forums pour obtenir de l’aide. Nous vous aiderons à mener votre requête à bien !

<div id="step-3-review-the-response">
  ### Étape 3. Examiner la réponse
</div>

Une fois que vous avez effectué une requête avec succès, vous recevrez un payload contenant des métadonnées liées à la requête.

Si vous avez utilisé un endpoint qui emploie la méthode HTTP GET, vous recevrez des métadonnées relatives à la ressource (Post, utilisateur, List, Space, etc.) visée par votre requête, au format JSON. Passez en revue les différents champs renvoyés et vérifiez si vous pouvez faire correspondre les informations demandées au contenu sur X.

Si vous avez utilisé un endpoint qui emploie une méthode HTTP POST, PUT ou DELETE, vous avez effectué une action sur X. Rendez-vous sur X.com ou sur l’application mobile et vérifiez si vous pouvez retrouver cette action.

<div id="step-4-adjust-your-request-using-parameters">
  ### Étape 4. Ajustez votre requête à l’aide de paramètres
</div>

Chaque endpoint propose un ensemble de paramètres différent que vous pouvez utiliser pour modifier votre requête. Par exemple, vous pouvez demander des champs de métadonnées supplémentaires lorsque vous utilisez des endpoints GET avec les paramètres fields et expansions. Vous pouvez aussi tester diverses options de filtrage avec des endpoints tels que [search Posts](/fr/x-api/posts/search/introduction), [Post counts](/fr/x-api/posts/counts/introduction) et [filtered stream](/fr/x-api/posts/filtered-stream/introduction) afin de restreindre les données reçues aux seuls Posts qui vous intéressent.

Vous trouverez la liste complète des paramètres de requête et des champs dans la référence de l’API de l’endpoint sur lequel vous travaillez, ainsi qu’une multitude d’autres informations utiles pour l’intégration dans nos guides d’intégration et nos pages de notions fondamentales.

Pour en savoir plus sur l’ensemble des ressources pédagogiques que nous avons mises à votre disposition, consultez notre guide [Important resources](/fr/x-api/getting-started/important-resources).
