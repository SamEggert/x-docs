<div id="step-one-sign-up-for-a-developer-account">
  ### Première étape : créer un compte développeur
</div>

Créer un compte développeur est simple et rapide ! Cliquez sur le bouton ci-dessous, répondez à quelques questions, et vous pourrez commencer à explorer et à développer avec l’API X v2 à l’aide de l’[accès Basic](https://developer.x.com/en/portal/products/basic).

Ensuite, au cours de l’onboarding, vous créerez un projet et une application développeur associée, qui vous fourniront un jeu d’identifiants à utiliser pour authentifier toutes vos requêtes vers l’API.

[S’inscrire](https://developer.x.com/en/portal/products/basic)

<div id="step-two-save-your-apps-key-and-tokens-and-keep-them-secure">
  ### Deuxième étape : Enregistrez la clé et les jetons de votre application et conservez-les en lieu sûr
</div>

Une fois que vous avez accès et avez créé un projet et une application, vous pourrez trouver ou générer les identifiants suivants dans votre application développeur :

* **[API Key and Secret](https://developer.x.com/resources/fundamentals/authentication/api-key-and-secret)** : en pratique, le nom d’utilisateur et le mot de passe de votre application. Vous les utiliserez pour authentifier les requêtes nécessitant le [contexte utilisateur OAuth 1.0a](/fr/resources/fundamentals/authentication), ou pour générer d’autres jetons tels que des jetons d’accès utilisateur (User Access Tokens) ou un jeton d’accès d’application (App Access Token).

* **[Access Token and Secret](https://developer.x.com/resources/fundamentals/authentication/obtaining-user-access-tokens)** : en général, les jetons d’accès représentent l’utilisateur pour le compte duquel vous effectuez la requête. Ceux que vous pouvez générer via le portail développeur représentent l’utilisateur propriétaire de l’application. Vous les utiliserez pour authentifier les requêtes nécessitant le [contexte utilisateur OAuth 1.0a](/fr/resources/fundamentals/authentication). Si vous souhaitez effectuer des requêtes pour le compte d’un autre utilisateur, vous devrez utiliser le flux OAuth à 3 jambes afin qu’il vous autorise.

* **[Client ID and Client Secret](/fr/resources/fundamentals/authentication#how-to-connect-to-endpoints-using-oauth-2-0-authorization-code-flow-with-pkce)** : ces identifiants sont utilisés pour obtenir un jeton d’accès utilisateur avec l’authentification OAuth 2.0. Comme avec OAuth 1.0a, les jetons d’accès utilisateur servent à authentifier des requêtes qui fournissent des informations privées de compte ou effectuent des actions pour le compte d’un autre compte, mais avec des portées fines pour un meilleur contrôle des accès de l’application cliente au compte utilisateur.

* **[App only Access Token](/fr/resources/fundamentals/authentication#bearer-token-also-known-as-app-only)** : vous utiliserez ce jeton pour des requêtes vers des endpoints renvoyant des informations publiquement disponibles sur X.

Comme ces clés et jetons n’expirent pas sauf régénération, nous vous recommandons de les enregistrer dans un emplacement sécurisé, tel qu’un gestionnaire de mots de passe, dès réception de vos identifiants.

> **Veuillez noter : vos clés et jetons ne s’affichent qu’une seule fois dans le [developer portal](/fr/resources/fundamentals/developer-portal). Il est donc important de stocker ces identifiants dans votre gestionnaire de mots de passe dès leur génération.**
>
> Si vous égarez ou oubliez vos clés et jetons, vous devrez les régénérer, ce qui crée de nouveaux identifiants et invalide les anciens. Vous devrez alors mettre à jour toute intégration que vous avez configurée avec vos identifiants précédents.
>
> En savoir plus sur nos [bonnes pratiques d’authentification](/fr/resources/fundamentals/authentication#authentication-best-practices).
