<div id="introduction">
  ## Introduction
</div>

La pagination est une fonctionnalité des endpoints de l’API X v2 qui renvoient plus de résultats qu’il n’est possible d’inclure dans une seule réponse. Dans ce cas, les données sont renvoyées sous forme d’une série de « pages ». La pagination désigne les méthodes permettant de demander par programmation l’ensemble des pages afin de récupérer l’intégralité du jeu de résultats. Tous les endpoints de l’API ne prennent pas en charge ni n’exigent la pagination, mais elle est souvent utilisée lorsque les jeux de résultats sont importants.

<div id="use-cases-for-pagination">
  ### Cas d’utilisation de la pagination
</div>

**Pour récupérer tous les résultats d’une requête :** Utilisez la pagination pour obtenir l’ensemble des données pertinentes liées à une requête et à ses paramètres. Elle est nécessaire lorsque le nombre de résultats correspondants dépasse la valeur de `max_results` pour une requête. Répéter les requêtes à l’aide de jetons de pagination vous permet de récupérer tous les résultats. Lorsqu’une réponse est renvoyée sans valeur `next_token`, vous pouvez considérer que tous les résultats ont été parcourus. La pagination ne doit pas être utilisée pour l’interrogation récurrente (polling). Pour obtenir les résultats les plus récents depuis une requête précédente, utilisez l’interrogation avec `since_id`.

**Pour parcourir les résultats d’une requête :** La pagination offre des options directionnelles pour naviguer dans les résultats d’une requête, en s’appuyant sur les valeurs `next_token` et `previous_token` des réponses. Ces jetons peuvent être fournis en tant que `pagination_token` dans la requête suivante pour passer à la page suivante ou précédente des résultats.

<div id="pagination-token-definitions">
  ### Définitions des jetons de pagination
</div>

* `next_token` - Chaîne opaque renvoyée dans l&#39;objet meta des réponses sur les points de terminaison qui prennent en charge la pagination. Indique que d&#39;autres résultats sont disponibles et peut être utilisée comme paramètre `pagination_token` dans la requête suivante pour renvoyer la page suivante de résultats. La dernière page de résultats ne contient pas de `next_token`.
  * `previous_token` - Chaîne opaque renvoyée dans l&#39;objet meta des réponses sur les points de terminaison qui prennent en charge la pagination. Indique qu&#39;une page précédente de résultats est disponible et peut être utilisée comme paramètre `pagination_token` dans la requête suivante pour renvoyer la page précédente de résultats.
  * `pagination_token` - Paramètre utilisé dans les requêtes de pagination. À définir sur la valeur de `next_token` pour la page suivante de résultats. À définir sur la valeur de `previous_token` pour la page précédente de résultats.

<div id="fundamentals-of-pagination">
  ### Principes fondamentaux de la pagination
</div>

* Les endpoints qui utilisent la pagination répondent à une requête initiale avec la première page de résultats et incluent un `next_token` dans l’objet meta de la réponse JSON si d’autres pages de résultats sont disponibles. Pour récupérer l’ensemble des résultats, répétez ce processus jusqu’à ce qu’aucun `next_token` ne soit présent dans la réponse.
  * Les résultats sont renvoyés par ordre antéchronologique. Cela vaut au sein de chaque page comme entre plusieurs pages :
    * Le premier Post de la première réponse sera le plus récent.
    * Le dernier Post de la dernière réponse sera le plus ancien.
  * Le paramètre de requête `max_results` vous permet de définir le nombre de Posts renvoyés par page de réponse. Des valeurs par défaut et maximale existent pour `max_results`.
  * Toute implémentation de pagination implique l’extraction de jetons depuis la charge utile de la réponse, qui peuvent être réutilisés dans des requêtes ultérieures.
  * Dans certaines circonstances, vous pouvez obtenir moins que `max_results` par page. Ne supposez pas que le nombre de résultats par page sera toujours égal à la valeur du paramètre `max_results`.
  * Les meilleurs moyens d’utiliser la pagination pour obtenir des résultats complets sont d’employer une logique de boucle dans le code d’intégration ou d’utiliser une [bibliothèque](/fr/x-api/tools-and-libraries/overview) compatible avec X API v2.
