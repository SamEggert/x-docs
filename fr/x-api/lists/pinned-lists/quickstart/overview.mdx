<div id="getting-started-with-the-pinned-list-endpoint-group">
  ## Bien démarrer avec le groupe d’endpoints des Listes épinglées
</div>

Cette brève présentation vous aidera à effectuer votre première requête vers les endpoints des Listes épinglées avec [Postman](/fr/tutorials/postman-getting-started).

Veuillez consulter notre dépôt GitHub [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code) si vous souhaitez voir des exemples de code dans différents langages.

<Note>
  ### Prérequis

  Pour suivre ce guide, vous aurez besoin d’un ensemble de [keys and tokens](/fr/resources/fundamentals/authentication) pour authentifier votre requête. Vous pouvez générer ces clés et jetons en suivant ces étapes :

  * [Inscrivez-vous pour obtenir un compte développeur](https://developer.x.com/en/apply-for-access) et attendez l’approbation.
  * Créez un [Project](/fr/resources/fundamentals/projects) et une [developer App](/fr/resources/fundamentals/developer-apps) associée dans le portail développeur.
  * Accédez à la page “Keys and tokens” de votre App pour générer les identifiants requis. Assurez-vous d’enregistrer tous les identifiants dans un emplacement sécurisé.
</Note>

Charger la collection Postman de X API v2

Pour importer la collection Postman de X API v2 dans votre espace de travail, cliquez sur le bouton suivant :

<div id="authenticate-your-request">
  #### Authentifiez votre requête
</div>

Pour effectuer une requête vers les endpoints **lookup**, vous pouvez utiliser [OAuth 1.0a User Context](/fr/resources/fundamentals/authentication), [OAuth 2.0 App-Only](/fr/resources/fundamentals/authentication#oauth-2-0) ou [OAuth 2.0 Authorization Code with PKCE](fr/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2). En revanche, pour les endpoints **manage**, seule l’authentification avec OAuth 1.0a User Context ou OAuth 2.0 Authorization Code with PKCE est possible.

Dans tous les cas, lorsque vous utilisez Postman, les clés et jetons d’authentification par défaut seront automatiquement renseignés dans vos requêtes, à condition que votre environnement soit correctement configuré.

Pour cela, sélectionnez l’environnement nommé “X API v2” (dans l’angle supérieur droit de Postman) et ajoutez vos clés et jetons dans les champs “initial value” et “current value” (en cliquant sur l’icône en forme d’œil à côté du menu déroulant de l’environnement). Ces clés incluent :

* API Key (aussi appelé Consumer Key)
* API Secret Key (aussi appelé Consumer Secret)
* OAuth 1.0a user Access Token
* OAuth 1.0a user Access Token Secret
* OAuth 2.0 App Access Token
* OAuth 2.0 Client Key (disponible uniquement si vous avez configuré les paramètres d’authentification utilisateur OAuth 2.0 dans les paramètres de votre App)
* OAuth 2.0 Client Secret (disponible uniquement si vous avez configuré les paramètres d’authentification utilisateur OAuth 2.0 dans les paramètres de votre App)
