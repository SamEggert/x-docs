<div id="manage-lists-standard-v11-compared-to-x-api-v2">
  ### Gérer les listes : Standard v1.1 comparé à X API v2
</div>

Si vous avez travaillé avec les endpoints standard v1.1 [POST lists/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-create), [POST lists/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-destroy), et [POST lists/update](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-update), ce guide a pour objectif de vous aider à comprendre les similitudes et les différences entre les endpoints de gestion des listes du standard v1.1 et de X API v2.

* **Similarités**
  * Authentification
* **Différences**
  * URL des endpoints
  * Exigences pour les apps et les projets
  * Méthodes HTTP
  * Limites de débit
  * Paramètres de requête

<div id="similarities">
  #### Similarités
</div>

**Authentification**

Les deux versions d’endpoints prennent en charge [OAuth 1.0a User Context](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication). Par conséquent, si vous utilisiez auparavant l’un des endpoints standard v1.1 pour gérer des listes, vous pouvez continuer à utiliser la même méthode d’authentification lors de la migration vers la version X API v2.

<div id="differences">
  #### Différences
</div>

**URL des endpoints**

* Endpoints Standard v1.1 :
  * POST https://api.x.com/1.1/lists/create.json
    (Crée une liste)
  * POST https://api.x.com/1.1/lists/destroy.json
    (Supprime une liste)
  * POST https://api.x.com/1.1/lists/update.json
    (Met à jour une liste)
* Endpoint X API v2 :
  * POST https://api.x.com/2/lists
    (Crée une liste)

  * DELETE https://api.x.com/2/lists/:id
    (Supprime une liste)

  * PUT https://api.x.com/2/lists/:id
    (Met à jour une liste)

**Limites de débit**

| **Standard v1.1** | **X API v2** |
| :--- | :--- |
| /1.1/lists/create.json<br /><br />aucune | /2/lists<br /><br />300 requêtes par fenêtre de 15 minutes avec OAuth 1.0a User Context |
| /1.1/lists/destroy.json<br /><br />aucune | /2/lists/:id<br /><br />300 requêtes par fenêtre de 15 minutes avec OAuth 1.0a User Context |
| /1.1/lists/update.json<br /><br />aucune | /2/lists/:id<br /><br />300 requêtes par fenêtre de 15 minutes avec OAuth 1.0a User Context |

**Exigences pour l’App et le Project**

Les endpoints X API v2 exigent que vous utilisiez des identifiants provenant d’une [developer App](/fr/resources/fundamentals/developer-apps) associée à un [Project](/fr/resources/fundamentals/projects) lors de l’authentification de vos requêtes. Tous les endpoints X API v1.1 peuvent utiliser des identifiants issus d’Apps autonomes ou d’Apps liées à un Project.

**Paramètres de requête**

Les paramètres de requête Standard v1.1 suivants ont des équivalents dans X API v2 :

**Créer une liste**

| **Standard** | **X API v2** |
| :--- | :--- |
| name | name |
| mode | private |
| description | description |

**Supprimer/Mettre à jour une liste**

| **Standard** | **X API v2** |
| :--- | :--- |
| owner&#95;screen&#95;name | Aucun équivalent |
| owner&#95;id | Aucun équivalent |
| list&#95;id | id  |
| slug | Aucun équivalent |

<Note>
  **Veuillez noter :** les paramètres Standard v1.1 sont passés en tant que paramètres de requête, tandis que les paramètres X API v2 sont passés en tant que paramètres de corps (pour l’endpoint POST) ou paramètres de chemin (pour les endpoints DELETE et PUT).
</Note>
