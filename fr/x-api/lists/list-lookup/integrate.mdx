<div id="authentication">
  #### Authentification
</div>

Tous les endpoints de X API v2 exigent que vous authentifiiez vos requêtes à l’aide d’un ensemble d’identifiants, également appelés clés et jetons. Vous pouvez utiliser OAuth 1.0a User Context, App only ou OAuth 2.0 Authorization Code with PKCE pour authentifier vos requêtes vers ces endpoints.

[OAuth 1.0a User Context](/fr/resources/fundamentals/authentication#oauth-1-0a-user-context) signifie que vous devez utiliser un ensemble d’API Keys et de user Access Tokens pour effectuer une requête réussie. Les access tokens doivent être associés à l’utilisateur pour le compte duquel vous effectuez la requête. Si vous souhaitez générer un ensemble d’Access Tokens pour un autre utilisateur, celui-ci doit autoriser votre App via le [3-legged OAuth flow](/fr/resources/fundamentals/authentication#obtaining-access-tokens-using-3-legged-oauth-flow).

Veuillez noter qu’OAuth 1.0a peut être difficile à utiliser. Si vous n’êtes pas familier avec cette méthode d’authentification, nous vous recommandons d’utiliser une [library](/fr/x-api/tools-and-libraries/overview), un outil comme Postman, ou d’utiliser OAuth 2.0 pour authentifier vos requêtes.

[OAuth 2.0 Authorization Code with PKCE](/fr/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2) offre un meilleur contrôle sur le scope d’une application et prend en charge des parcours d’autorisation sur plusieurs appareils. OAuth 2.0 vous permet de sélectionner des scopes fins et granulaires, qui vous accordent des permissions spécifiques au nom d’un utilisateur.

Pour activer OAuth 2.0 dans votre App, vous devez l’activer dans les paramètres d’authentification de votre App, situés dans la section App settings du developer portal.

[App only](/fr/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) requiert simplement d’inclure un [App only Access Token](/fr/resources/fundamentals/authentication#app-only-authentication-and-oauth-2-0-bearer-token) dans votre requête. Vous pouvez générer un App only token directement depuis une developer App, ou en générer un via l’endpoint [POST oauth2/token](/fr/resources/fundamentals/authentication#post-oauth2-token).

<div id="developer-portal-projects-and-developer-apps">
  #### Portail développeur, Projets et applications développeur
</div>

Pour obtenir un ensemble d’identifiants d’authentification compatibles avec les endpoints de X API v2, vous devez [créer un compte développeur](https://developer.x.com/en/portal/petition/essential/basic-info), configurer un [Projet](/fr/resources/fundamentals/projects) dans ce compte, puis créer une [application développeur](/fr/resources/fundamentals/developer-apps) dans ce Projet. Vous pourrez ensuite retrouver vos clés et jetons dans votre application développeur.
 

<div id="rate-limits">
  #### Limites de débit
</div>

Chaque jour, des milliers de développeurs envoient des requêtes vers l’API X. Pour gérer le volume important de ces requêtes, des [limites de débit](/fr/x-api/fundamentals/rate-limits) sont appliquées à chaque endpoint, ce qui limite le nombre de requêtes que vous pouvez effectuer au nom de votre application ou d’un utilisateur authentifié.

Ces endpoints sont soumis à des limites à la fois au niveau de l’application et de l’utilisateur. La limite au niveau de l’application signifie que vous, en tant que développeur, ne pouvez effectuer qu’un certain nombre de requêtes vers cet endpoint sur une période donnée depuis n’importe quelle application (en utilisant soit l’API Key et l’API Secret Key, soit le App-only Access Token). La limite au niveau de l’utilisateur signifie que l’utilisateur authentifié pour lequel vous effectuez la requête ne peut effectuer une recherche de liste qu’un certain nombre de fois, et ce, depuis n’importe quelle application de développeur.

Le tableau ci-dessous présente les limites de débit pour chaque endpoint.

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **Méthode HTTP** | **Limite de débit** |
| /2/lists/:id | GET | 75 requêtes par 15 minutes |
| /2/users/:id/owned&#95;lists | GET | 15 requêtes par 15 minutes |

Champs et expansions

L’endpoint GET de l’API X v2 permet aux utilisateurs de sélectionner exactement les données à récupérer depuis l’API à l’aide d’outils appelés `fields` et `expansions`. Le paramètre `expansions` permet d’étendre les objets référencés dans la charge utile. Par exemple, rechercher une liste par ID permet de récupérer les [expansions](/fr/x-api/fundamentals/expansions) suivantes :

* `owner_id`

Le paramètre `fields` permet de sélectionner précisément quels [champs](/fr/x-api/fundamentals/fields), au sein des différents objets de données, vous souhaitez recevoir. Ces endpoints renvoient principalement des objets List. Par défaut, l’objet List renvoie les champs `id` et `name`. Pour recevoir des champs supplémentaires tels que `list.created_at` ou `list.follower_count`, vous devez les demander explicitement à l’aide du paramètre list.fields.

Nous avons ajouté à notre [dictionnaire de données de l’API X v2](/fr/x-api/fundamentals/data-dictionary) un guide sur l’utilisation conjointe des [fields et expansions](/fr/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions).

Le tableau ci-dessous présente les champs et expansions disponibles pour ce groupe d’endpoints :

|     |     |     |
| :--- | :--- | :--- |
| **Endpoint** | **Champs** | **Expansions** |
| /2/lists/:id | `list.fields`<br /><br />`user.fields` | `owner_id` |
| /2/users/:id/owned&#95;lists | `list.fields`<br /><br />`user.fields` | `owner_id` |

<div id="pagination">
  #### Pagination
</div>

La recherche des listes détenues par un utilisateur peut renvoyer un volume important de données. Pour garantir des résultats cohérents et performants à tout moment, nous utilisons la pagination. La pagination est une fonctionnalité des points de terminaison de X API v2 qui renvoient plus de résultats qu’une seule réponse ne peut contenir. Dans ce cas, les données sont renvoyées sous forme de « pages ». Pour en savoir plus sur la manière de [parcourir les résultats par pages.](/fr/x-api/fundamentals/pagination)
