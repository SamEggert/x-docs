<div id="manage-list-members-standard-v11-compared-to-x-api-v2">
  ### Gérer les membres de listes : standard v1.1 comparé à X API v2
</div>

Si vous avez utilisé les endpoints standard v1.1 [POST lists/members/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-create) et [POST lists/members/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-destroy), ce guide a pour objectif de vous aider à comprendre les similitudes et les différences entre les endpoints de gestion des membres de listes du standard v1.1 et de X API v2.

* **Similarités**
  * Authentification
* **Différences**
  * URL des endpoints
  * Exigences pour les applications et les projets
  * Méthodes HTTP
  * Limites de débit
  * Paramètres de requête

<div id="similarities">
  #### Similitudes
</div>

**Authentification**

Les deux versions d’endpoint prennent en charge [OAuth 1.0a User Context](https://developer.x.com/content/developer-twitter/resources/fundamentals/authentication). Ainsi, si vous utilisiez auparavant l’un des endpoints standard v1.1 pour gérer les membres de listes, vous pouvez continuer à utiliser la même méthode d’authentification lors de votre migration vers la version X API v2.

<div id="differences">
  #### Différences
</div>

**URL d’endpoint**

* Endpoints standard v1.1 :
  * POST https://api.x.com/1.1/lists/members/create.json
    (Ajoute un membre à une liste spécifiée)
  * POST https://api.x.com/1.1/lists/members/destroy.json
    (Supprime un membre d’une liste spécifiée)
* Endpoint X API v2 :
  * POST https://api.x.com/2/lists/:id/members
    (Ajoute un membre à une liste spécifiée)

  * DELETE https://api.x.com/2/lists/:id/members/:user_id
    (Supprime un membre d’une liste spécifiée)

**Limites de débit**

| **Standard v1.1** | **X API v2** |
| :--- | :--- |
| /1.1/lists/members/create.json<br /><br />aucune | /2/lists/:id/members<br /><br />300 requêtes par fenêtre de 15 minutes avec OAuth 1.0a (User Context)<br /><br />300 requêtes par fenêtre de 15 minutes avec OAuth 2.0 (Authorization Code avec PKCE) |
| /1.1/lists/members/destroy.json<br /><br />aucune | /2/lists/:id/members/:user_id<br /><br />300 requêtes par fenêtre de 15 minutes avec OAuth 1.0a (User Context)<br /><br />300 requêtes par fenêtre de 15 minutes avec OAuth 2.0 (Authorization Code avec PKCE) |

**Exigences relatives aux Apps et aux Projects**

Les endpoints X API v2 exigent que vous utilisiez des identifiants d’une [developer App](/fr/resources/fundamentals/developer-apps) associée à un [Project](/fr/resources/fundamentals/projects) lors de l’authentification de vos requêtes. Tous les endpoints X API v1.1 peuvent utiliser des identifiants provenant d’Apps autonomes ou d’Apps liées à un Project.

**Paramètres de requête**

Les paramètres de requête standard v1.1 suivants ont des équivalents dans X API v2 :

| **Standard v1.1** | **X API v2** |
| :--- | :--- |
| list_id | id  |
| slug | Aucun équivalent |
| screen_name | Aucun équivalent |
| owner_screen_name | Aucun équivalent |
| owner_id | Aucun équivalent |
