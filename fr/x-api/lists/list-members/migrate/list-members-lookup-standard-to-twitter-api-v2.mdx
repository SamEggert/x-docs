<div id="list-members-lookup-standard-v11-compared-to-x-api-v2">
  ### Recherche de membres de liste : Standard v1.1 comparé à X API v2
</div>

Si vous avez travaillé avec les points de terminaison standard v1.1 [GET lists/members](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-members) et [GET lists/memberships](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-memberships), ce guide a pour but de vous aider à comprendre les similitudes et les différences entre les points de terminaison des membres de liste du standard v1.1 et de X API v2.

* **Similarités**
  * Méthodes d’authentification
* **Différences**
  * URL des points de terminaison
  * Limites de débit
  * Exigences relatives aux applications et aux projets
  * Limites du nombre d’objets de données par requête
  * Formats des données de réponse
  * Paramètres de requête

<div id="similarities">
  #### Similarités
</div>

**Authentification**

Les deux versions d’endpoints prennent en charge à la fois [OAuth 1.0a (contexte utilisateur)](/fr/resources/fundamentals/authentication#oauth-1-0a-2) et l’authentification [App only](/fr/resources/fundamentals/authentication#oauth-2-0). Par conséquent, si vous utilisiez auparavant l’un des endpoints standard v1.1 pour les membres de liste, vous pouvez continuer à utiliser la même méthode d’authentification en migrant vers la version X API v2.

Selon la bibliothèque ou le package d’authentification que vous utilisez, l’authentification App only est probablement le moyen le plus simple de démarrer et peut être configurée avec un simple en-tête de requête. Pour savoir comment générer un jeton d’accès App only, consultez [ce guide App only](/fr/resources/fundamentals/authentication#bearer-token-also-known-as-app-only).

<div id="differences">
  #### Différences
</div>

**URLs des endpoints**

* Endpoints standard v1.1 :
  * GET https://api.x.com/1.1/lists/members.json
    (Récupérer les membres d&#39;une Liste spécifiée)
  * GET https://api.x.com/1.1/lists/memberships.json
    (Récupérer les Listes dont un utilisateur est membre)
* Endpoint X API v2 :
  * GET https://api.x.com/2/lists/:id/members
    (Récupérer les membres d&#39;une Liste spécifiée)

  * GET https://api.x.com/2/users/:id/list&#95;memberships
    (Récupérer les Listes dont un utilisateur est membre)

**Limites de débit**

|     |     |
| :--- | :--- |
| **Standard v1.1** | **X API v2** |
| /1.1/lists/members.json<br /><br />900 requêtes par fenêtre de 15 minutes avec OAuth 1.0a User Context<br /><br />15 requêtes par fenêtre de 15 minutes avec App only | /2/lists/:id/members<br /><br />900 requêtes par fenêtre de 15 minutes avec OAuth 1.0a User Context<br /><br />900 requêtes par fenêtre de 15 minutes avec OAuth 2.0 Authorization Code with PKCE<br /><br />900 requêtes par fenêtre de 15 minutes avec App only |
| /1.1/lists/memberships.json<br /><br />15 requêtes par fenêtre de 15 minutes avec OAuth 1.0a User Context<br /><br />15 requêtes par fenêtre de 15 minutes avec App only | /2/users/:id/list&#95;memberships<br /><br />15 requêtes par fenêtre de 15 minutes avec OAuth 1.0a User Context<br /><br />15 requêtes par fenêtre de 15 minutes avec OAuth 2.0 Authorization Code with PKCE<br /><br />15 requêtes par fenêtre de 15 minutes avec App only |

**Exigences relatives aux Apps et aux Projects**

Les endpoints X API v2 exigent que vous utilisiez des identifiants provenant d&#39;une [developer App](/fr/resources/fundamentals/developer-apps) associée à un [Project](/fr/resources/fundamentals/projects) lors de l&#39;authentification de vos requêtes. Tous les endpoints X API v1.1 peuvent utiliser des identifiants provenant d&#39;Apps autonomes ou d&#39;Apps associées à un Project.

**Limites d&#39;objets de données par requête**

L&#39;endpoint standard v1.1 /1.1/lists/members permet de renvoyer jusqu&#39;à 5000 utilisateurs par requête. Les nouveaux endpoints v2 permettent de renvoyer jusqu&#39;à 100 utilisateurs par requête. Par défaut, 100 objets utilisateur sont renvoyés ; pour modifier le nombre de résultats, vous devez passer un paramètre de requête max&#95;results= avec une valeur entre 1 et 100 ; vous pouvez ensuite passer le next&#95;token renvoyé dans la charge utile de la réponse au paramètre de requête pagination&#95;token dans votre requête suivante.

De plus, l&#39;endpoint /1.1/lists/memberships permet de renvoyer jusqu&#39;à 1000 Listes par requête. Avec le remplacement v2, l&#39;endpoint autorise jusqu&#39;à 100 Listes par requête. Par défaut, 100 objets Liste sont renvoyés ; utilisez les paramètres de requête max&#95;results= et pagination&#95;token de la même manière que pour /1.1/lists/members afin de modifier le nombre de résultats.

**Format des données de réponse**

L&#39;une des principales différences entre les versions d&#39;endpoints standard v1.1 et X API v2 est la manière de sélectionner les champs renvoyés dans la charge utile.

Pour les endpoints standard, de nombreux champs de réponse sont renvoyés par défaut, puis vous pouvez utiliser des paramètres pour indiquer les champs supplémentaires ou les ensembles de champs à inclure dans la charge utile.

La version X API v2 /users/:id/list&#95;memberships fournit par défaut les champs id et name de la Liste. Pour demander des champs ou des objets supplémentaires, vous devez utiliser les paramètres [fields](/fr/x-api/fundamentals/fields) et [expansions](/fr/x-api/fundamentals/expansions). Tous les champs de Liste demandés via cet endpoint sont renvoyés dans l&#39;objet principal List. Tout objet étendu et ses champs sont renvoyés dans un objet includes au sein de la réponse. Vous pouvez ensuite faire correspondre tout objet étendu à l&#39;objet principal List en faisant correspondre les IDs présents à la fois dans l&#39;objet principal et dans l&#39;objet étendu.

Voici des exemples de champs et d&#39;expansions possibles pour les Listes :

* created&#95;at

* follower&#95;count

* member&#95;count

* owner&#95;id

* description

* private

|     |     |
| :--- | :--- |
| **Endpoint** | **Expansion** |
| /2/lists/:id/members | pinned&#95;tweet&#95;id |
| /2/users/:id/list&#95;memberships | owner&#95;id |

Nous vous encourageons à en savoir plus sur ces nouveaux paramètres dans leurs guides respectifs ou en consultant notre guide sur [how to use fields and expansions](/fr/x-api/fundamentals/data-dictionary#how-to-use-fields-and-expansions). 

Nous avons également préparé un [guide de migration du format des données](/fr/x-api/migrate/data-format-migration) pour vous aider à faire correspondre les champs standard v1.1 aux nouveaux champs v2. Ce guide vous indiquera également l’expansion et le paramètre de champ précis à transmettre dans votre requête v2 pour obtenir des champs spécifiques. 

En plus des changements dans la façon de demander certains champs, X API v2 introduit également de nouveaux schémas JSON pour les objets renvoyés par les API, notamment les objets [Post](/fr/x-api/fundamentals/data-dictionary#tweet) et [user](/fr/x-api/fundamentals/data-dictionary#user).

* Au niveau racine du JSON, les endpoints standard renvoient des objets Post dans un tableau statuses, tandis que X API v2 renvoie un tableau data. 

* Au lieu de faire référence à des &quot;statuses&quot; Retweeted et Quoted, le JSON de X API v2 fait référence à des Tweets Retweeted et Quoted. De nombreux champs hérités et obsolètes, tels que contributors et user.translator&#95;type, sont supprimés. 

* Au lieu d’utiliser à la fois favorites (dans l’objet Post) et favourites (dans l’objet user), X API v2 utilise le terme like. 

* X adopte la convention selon laquelle les valeurs JSON sans valeur (par exemple null) ne sont pas incluses dans la charge utile. Les attributs Post et user ne sont inclus que s’ils ont des valeurs non nulles.

**Paramètres de requête**

Les paramètres de requête standard v1.1 suivants ont des équivalents dans X API v2 :

**Recherche des membres d’une liste**

|     |     |
| :--- | :--- |
| **Standard v1.1** | **X API v2** |
| list&#95;id | id  |
| slug | Aucun équivalent |
| owner&#95;screen&#95;name | Aucun équivalent |
| owner&#95;id | Aucun équivalent |
| count | max&#95;results |
| cursor | pagination&#95;token |
| include&#95;entities | Aucun équivalent |
| skip&#95;status | Aucun équivalent |

**Recherche d’appartenance à une liste**

|     |     |
| :--- | :--- |
| **Standard v1.1** | **X API v2** |
| user&#95;id | id  |
| screen&#95;name | Aucun équivalent |
| count | max&#95;results |
| cursor | pagination&#95;token |
