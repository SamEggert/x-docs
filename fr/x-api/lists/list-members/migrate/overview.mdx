<div id="comparing-x-apis-list-members-endpoints">
  ## Comparaison des endpoints List members de l’API X
</div>

Le groupe d’endpoints List members de la v2 remplacera les endpoints standard de la v1.1 [GET lists/members](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-members), [GET lists/memberships](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-memberships), [POST lists/members/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-create) et [POST lists/members/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-destroy). Si vous avez du code, des applications ou des outils qui utilisent l’une de ces versions des endpoints List members et que vous envisagez de migrer vers le nouvel endpoint de l’API X v2, cet ensemble de guides est fait pour vous. 

<div id="list-members-lookup">
  ### Recherche des membres de liste
</div>

Les endpoints de recherche des membres de liste v2 remplaceront les endpoints standard [GET lists/members](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-members), [GET lists/memberships](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-memberships).

Les tableaux suivants comparent les endpoints de liste de la v1.1 standard et de X API v2 :

**Recherche de membres de liste**

|     |     |     |
| :--- | :--- | :--- |
| Description | v1.1 standard | X API v2 |
| Méthodes HTTP prises en charge | `GET` | `GET` |
| Domaine d’hébergement | `https://api.x.com` | `https://api.x.com` |
| Chemin d’endpoint | `/1.1/lists/members.json` | `/2/lists/:id/members` |
| [Authentication](/fr/resources/fundamentals/authentication) | OAuth 1.0a contexte utilisateur<br /><br />App only | OAuth 1.0a contexte utilisateur<br /><br />OAuth 2.0 Authorization Code with PKCE<br /><br />App only |
| [Rate limits](/fr/resources/fundamentals/rate-limits) de requêtes par défaut | 900 requêtes par 15 min avec OAuth 1.0a<br /><br />75 requêtes par 15 min avec App only | 900 requêtes par 15 min avec OAuth 1.0a<br /><br />900 requêtes par 15 min avec OAuth 2.0<br /><br />900 requêtes par 15 min avec App only |

**Recherche d’appartenances à des listes**

|     |     |     |
| :--- | :--- | :--- |
| Description | v1.1 standard | X API v2 |
| Méthodes HTTP prises en charge | `GET` | `GET` |
| Domaine d’hébergement | `https://api.x.com` | `https://api.x.com` |
| Chemin d’endpoint | `/1.1/lists/memberships.json` | `/2/users/:id/list_memberships` |
| [Authentication](/fr/resources/fundamentals/authentication) | OAuth 1.0a contexte utilisateur<br /><br />App only | OAuth 1.0a contexte utilisateur<br /><br />OAuth 2.0 Authorization Code with PKCE<br /><br />App only |
| [Rate limits](/fr/resources/fundamentals/rate-limits) de requêtes par défaut | 75 requêtes par 15 min avec OAuth 1.0a<br /><br />75 requêtes par 15 min avec App only | 75 requêtes par 15 min avec OAuth 1.0a<br /><br />75 requêtes par 15 min avec OAuth 2.0<br /><br />75 requêtes par 15 min avec App only |

<div id="manage-list-members">
  ### Gérer les membres de List
</div>

Les endpoints v2 de gestion des membres de List remplaceront les endpoints standard [POST lists/members/create](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-create) et [POST lists/members/destroy](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/post-lists-members-destroy).

Les tableaux suivants comparent les endpoints standard v1.1 et X API v2 pour List :

**Ajouter un membre**

| Description | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| Méthodes HTTP prises en charge | POST | POST |
| Domaine hôte | https://api.x.com | https://api.x.com |
| Chemin d’endpoint | /1.1/lists/members/create.json | /2/lists/:id/members |
| [Authentication](/fr/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code with PKCE |
| [Limites de débit](/fr/resources/fundamentals/rate-limits) par défaut | Aucune | 300 requêtes par 15 min (par utilisateur) |

**Supprimer un membre**

| Description | Standard v1.1 | X API v2 |
| :--- | :--- | :--- |
| Méthodes HTTP prises en charge | POST | DELETE |
| Domaine hôte | https://api.x.com | https://api.x.com |
| Chemin d’endpoint | /1.1/lists/members/destroy.json | /2/lists/:id/:user&#95;id |
| [Authentication](/fr/resources/fundamentals/authentication) | OAuth 1.0a User Context | OAuth 1.0a User Context<br /><br />OAuth 2.0 Authorization Code with PKCE |
| [Limites de débit](/fr/resources/fundamentals/rate-limits) par défaut | Aucune | 300 requêtes par 15 min (par utilisateur) |

Pour accéder aux endpoints X API v2, vous devez [vous inscrire à un compte développeur](https://developer.x.com/en/portal/petition/essential/basic-info). Lors de l’authentification, vous devez utiliser des clés et des jetons issus d’une [developer App](/fr/resources/fundamentals/developer-apps) située au sein d’un [Project](/fr/resources/fundamentals/projects).

Pour en savoir plus sur l’accès aux endpoints X API v2, consultez notre page [getting started](/fr/x-api/getting-started/getting-access).
