<div id="steps-to-build-a-list-members-lookup-request">
  ### Étapes pour créer une requête de recherche des membres d’une Liste
</div>

**Première étape : choisir la collection d’endpoints List dans Postman**

Une fois la collection X API v2 chargée dans Postman, accédez au dossier “List”, sélectionnez le sous-dossier “List members”, puis choisissez &quot;Members lookup&quot;.
 

**Deuxième étape : identifier et indiquer la Liste dont vous souhaitez récupérer les membres**

Vous devez préciser la Liste dont vous souhaitez obtenir les membres. Vous pouvez trouver l’ID de la Liste en vous rendant sur x.com, en cliquant sur une Liste, puis en regardant l’URL. Par exemple, l’ID de Liste de l’URL suivante est 84839422.

https://x.com/i/lists/84839422

L’ID cible peut être n’importe quel ID de Liste valide. Dans Postman, accédez à l’onglet &quot;Params&quot; et saisissez votre ID dans la colonne &quot;Value&quot; de la variable de chemin id. Veillez à ne pas inclure d’espaces avant ou après l’ID.

|     |     |
| :--- | :--- |
| **Key** | **Value** |
| `id` | 84839422 (List ID) |

**Troisième étape : identifier et indiquer les champs que vous souhaitez récupérer**

Si vous cliquez sur le bouton &quot;Send&quot; après cette étape, vous recevrez dans la réponse les champs par défaut du [user object](/fr/x-api/fundamentals/data-dictionary#user) : id, name et username.

Si vous souhaitez recevoir d’autres champs au-delà de id, name et username, vous devrez les spécifier dans votre requête à l’aide des paramètres [field](/fr/x-api/fundamentals/fields) et/ou [expansion](/fr/x-api/fundamentals/expansions).

Pour cet exercice, nous allons demander trois ensembles supplémentaires de champs provenant d’objets différents :

* Le champ supplémentaire user.created_at dans les objets utilisateur principaux.
* Le [Post object](/fr/x-api/fundamentals/data-dictionary#tweet) complet en utilisant le paramètre expansion.
* Le champ supplémentaire tweet.created_at dans les objets Post associés.

Dans Postman, accédez à l’onglet &quot;Params&quot; et ajoutez la paire clé:valeur suivante au tableau &quot;Query Params&quot; :

|     |     |     |
| :--- | :--- | :--- |
| **Key** | **Value** | **Returned fields** |
| user.fields | created_at | `created_at` |
| expansions | pinned_tweet_id | `includes.tweets.id,  <br/>includes.tweets.text  <br/>` |
| tweet.fields | created_at | `includes.tweets.created_at` |

Vous devriez maintenant voir une URL similaire à côté du bouton “Send” :

`https://api.x.com/2/lists/84839422/members?user.fields=created_at&expansions=pinned_tweet_id&tweet.fields=created_at`
