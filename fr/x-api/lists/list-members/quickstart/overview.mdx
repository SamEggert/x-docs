<div id="getting-started-with-the-list-members-endpoint-group">
  ## Premiers pas avec le groupe d’endpoints List members
</div>

Cette présentation rapide vous aidera à effectuer votre première requête vers les endpoints List members à l’aide de [Postman](/fr/tutorials/postman-getting-started).

Si vous souhaitez voir des exemples de code dans différents langages, veuillez consulter notre dépôt GitHub [X API v2 sample code](https://github.com/xdevplatform/Twitter-API-v2-sample-code).

<Note>
  **Prérequis**

  Pour suivre ce guide, vous aurez besoin d’un jeu de [keys and tokens](/fr/resources/fundamentals/authentication) pour authentifier votre requête. Vous pouvez générer ces clés et jetons en suivant ces étapes :

  * [Sign up for a developer account](https://developer.x.com/en/apply-for-access) et obtenir l’approbation.
  * Créez un [Project](/fr/resources/fundamentals/projects) et une [developer App](/fr/resources/fundamentals/developer-apps) associés dans le portail développeur.
  * Accédez à la page « Keys and tokens » de votre App pour générer les identifiants requis. Assurez-vous d’enregistrer tous les identifiants dans un emplacement sécurisé.
</Note>

<div id="authenticate-your-request">
  #### Authentifiez votre requête
</div>

Pour envoyer une requête réussie vers les endpoints de **lookup**, vous pouvez utiliser [OAuth 1.0a User Context](/fr/resources/fundamentals/authentication#oauth-1-0a-2), [App only](/fr/resources/fundamentals/authentication#bearer-token-also-known-as-app-only) ou [OAuth 2.0 Authorization Code with PKCE](fr/resources/fundamentals/authentication#oauth-2-0-authorization-code-flow-with-pkce-2). En revanche, pour les endpoints de **manage**, l’authentification est possible uniquement avec OAuth 1.0a User Context ou OAuth 2.0 Authorization Code with PKCE.

Dans Postman, les clés et jetons d’authentification par défaut sont automatiquement renseignés dans vos requêtes dès lors que votre environnement est correctement configuré.

Pour ce faire, sélectionnez l’environnement nommé « X API v2 » (en haut à droite de Postman) et ajoutez vos clés et jetons dans les champs « initial value » et « current value » (en cliquant sur l’icône en forme d’œil à côté du menu déroulant de l’environnement). Ces clés incluent :

* API Key (également appelé Consumer Key)
* API Secret Key (également appelé Consumer Secret)
* OAuth 1.0a user Access Token
* OAuth 1.0a user Access Token Secret
* App only Access Token
* OAuth 2.0 Client Key (disponible uniquement si vous avez configuré les paramètres OAuth 2.0 User Authentication dans les paramètres de votre application)
* OAuth 2.0 Client Secret (disponible uniquement si vous avez configuré les paramètres OAuth 2.0 User Authentication dans les paramètres de votre application)
