  ## Comparaison des endpoints de recherche de publications de listes de l’API X
</div>

L’endpoint v2 de recherche de publications de listes remplacera l’endpoint standard v1.1 [GET lists/statuses](https://developer.x.com/en/docs/twitter-api/v1/accounts-and-users/create-manage-lists/api-reference/get-lists-statuses). Si vous avez du code, des applications ou des outils qui utilisent cette version de l’endpoint et que vous envisagez de migrer vers le nouvel endpoint X API v2, cet ensemble de guides est fait pour vous.

Les tableaux suivants comparent les endpoints de listes standard v1.1 et X API v2 :

**Recherche de liste par ID**

|     |     |     |
| :--- | :--- | :--- |
| Description | v1.1 standard | X API v2 |
| Méthodes HTTP prises en charge | `GET` | `GET` |
| Domaine hôte | `https://api.x.com` | `https://api.x.com` |
| Chemin de l’endpoint | `/1.1/lists/statuses.json` | `/2/lists/:id/tweets` |
| [Authentication](/fr/resources/fundamentals/authentication) | Contexte utilisateur OAuth 1.0a<br /><br />Application uniquement | Contexte utilisateur OAuth 1.0a<br /><br />OAuth 2.0 Authorization Code avec PKCE<br /><br />Application uniquement |
| [Rate limits](/fr/resources/fundamentals/rate-limits) de requêtes par défaut | 900 requêtes par 15 min avec OAuth 1.0a<br /><br />900 requêtes par 15 min avec application uniquement | 900 requêtes par 15 min avec OAuth 1.0a<br /><br />900 requêtes par 15 min avec OAuth 2.0<br /><br />900 requêtes par 15 min avec application uniquement |

Pour accéder aux endpoints X API v2, vous devez [vous inscrire pour obtenir un compte développeur](https://developer.x.com/en/portal/petition/essential/basic-info). Lors de l’authentification, vous devez utiliser des clés et des jetons provenant d’une [application développeur](/fr/resources/fundamentals/developer-apps) située dans un [projet](/fr/resources/fundamentals/projects).

Pour en savoir plus sur l’accès aux endpoints X API v2, consultez notre page [prise en main](/fr/x-api/getting-started/getting-access).
