Les connexions TLS sont requises pour accéder aux points de terminaison de l’API X. La communication via TLS préserve la confidentialité et la sécurité des utilisateurs en protégeant les informations échangées entre l’utilisateur et l’API X lorsqu’elles transitent sur l’internet public. Les connexions à l’API X exigent la version 1.2 de TLS.


<div id="verification">
  ## Vérification
</div>

<div id="use-an-up-to-date-root-store">
  ### Utiliser un magasin de racines à jour
</div>

Il est important que votre application ou bibliothèque utilise un magasin de racines fiable et à jour pour vérifier le certificat X. Lorsque c’est possible, utiliser le magasin de racines fourni par votre système d’exploitation est généralement l’approche la plus simple. À défaut, le [magasin de racines Mozilla (NSS)](https://www.mozilla.org/en-US/about/governance/policies/security-group/certs/) est bien maintenu, de manière publique et transparente. curl propose également [une version de ce magasin au format PEM](https://curl.haxx.se/docs/caextract.html).

X émet actuellement la majorité de ses certificats depuis le [DigiCert High Assurance EV Root CA](https://www.digicert.com/digicert-root-certificates.htm), mais ce n’est pas le cas de 100 % des certificats liés à X et cela pourrait ne pas rester vrai indéfiniment ; se fier uniquement aux autorités racines DigiCert actuellement utilisées pourrait donc entraîner des problèmes avec votre application à l’avenir.

<div id="check-crls-and-the-ocsp-status">
  ### Vérifier les CRL et l’état OCSP[](#check-crls-and-the-ocsp-status "Permalink to this headline")
</div>

De nombreuses applications ne consultent pas la liste de révocation de certificats (CRL) pour les certificats reçus ou s’en remettent au système d’exploitation pour le faire. Assurez-vous que votre application ou votre bibliothèque TLS est configurée pour imposer la vérification CRL et OCSP (Online Certificate Status Protocol) avant d’accepter le certificat de X.

<div id="cdns">
  ### CDNs[](#cdns "Permalink to this headline")
</div>

Pour l’affichage de Tweets contenant des médias, utilisez l’attribut `media_url_https` pour les URL HTTPS des images. À l’avenir, toutes les URL renvoyées par les points de terminaison de l’API fourniront des chemins HTTPS.

<div id="provide-an-indication-of-security-status">
  ## Indiquer l’état de sécurité
</div>

Dans la mesure du possible, affichez une indication de l’état actuel de la connexion entre votre application et X. Certains navigateurs l’indiquent à l’aide d’une icône de cadenas, tandis que d’autres affichent l’état de la connexion avec un message descriptif.
