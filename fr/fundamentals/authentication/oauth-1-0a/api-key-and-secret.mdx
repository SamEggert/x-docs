<div id="api-key-and-secret">
  ### Clé API et secret
</div>

La clé API et le secret (également appelés Consumer Key et Consumer Secret) sont les identifiants essentiels nécessaires pour accéder à l’API X. Ils font office de nom d’utilisateur et de mot de passe pour votre application X et permettent à l’API X d’identifier l’application à l’origine des requêtes.

Ces identifiants peuvent être utilisés par les [points de terminaison d’authentification](/fr/resources/fundamentals/authentication/api-reference) pour générer d’autres identifiants, tels que les [jetons d’accès utilisateur et leurs secrets](/fr/resources/fundamentals/authentication/oauth-1-0a/api-key-and-secret), ainsi que les [jetons Bearer](/fr/resources/fundamentals/authentication/oauth-2-0/bearer-tokens). Vous devez également utiliser ces identifiants avec des jetons d’accès et d’autres paramètres d’autorisation pour [autoriser les requêtes](/fr/resources/fundamentals/authentication/oauth-1-0a/authorizing-a-request) nécessitant l’authentification OAuth 1.0a en contexte utilisateur. 

<div id="how-to-acquire-an-api-key-and-secret">
  #### Comment obtenir une clé et un secret d’API
</div>

Pour obtenir une clé et un secret d’API X, veuillez suivre ces étapes :

1. [Créez un compte développeur X](https://developer.x.com/en/apply-for-access)
2. Créez une [X App](/fr/resources/fundamentals/developer-apps) dans le [developer portal](/fr/resources/fundamentals/developer-portal). Notez que si vous souhaitez utiliser [X API v2](/fr/x-api/introduction), vous devez ajouter votre X App à un [Project](/fr/resources/fundamentals/projects).
    

Lorsque vous créez votre X App, votre clé et votre secret d’API vous sont fournis, ainsi qu’un Bearer Token. Veuillez noter que nous n’affichons ces identifiants qu’une seule fois ; veillez donc à les enregistrer dans votre gestionnaire de mots de passe ou dans un emplacement sécurisé.

Vous trouverez d’autres recommandations sur la gestion de vos clés et tokens sur notre page [authentication best practices](/fr/resources/fundamentals/authentication/guides/authentication-best-practices), notamment des indications sur la marche à suivre si vos identifiants ont été compromis.
 

<div id="how-to-find-and-regenerate-your-api-key-and-secret-after-app-creation">
  #### Comment trouver et régénérer votre clé API et votre secret après la création d’une application
</div>

Si vous avez déjà créé une application et que vous devez retrouver ou régénérer votre clé API et votre secret, veuillez suivre ces étapes :

1. Accédez au portail développeur
2. Déployez le menu « Projects and Apps » dans la barre latérale
3. Ouvrez l’application associée à la clé API et au secret que vous souhaitez retrouver ou régénérer
4. Accédez à l’onglet « Keys and tokens »

Vous y trouverez toutes les informations d’identification associées à votre application. 
 

<div id="how-to-use-your-api-key-and-secret">
  #### Comment utiliser votre clé et votre secret d’API
</div>

Si vous faites simplement vos premiers pas sur la X Developer Platform, nous vous recommandons d’utiliser un [outil ou une bibliothèque](/fr/resources/tools-and-libraries) pour découvrir ce que la plateforme propose. Ces outils gèrent l’authentification de manière fluide et peuvent vous faire gagner beaucoup de temps et vous éviter bien des frustrations. Nous recommandons particulièrement de [commencer avec Postman](/fr/tutorials/postman-getting-started) ou [Insomnia](https://insomnia.rest/) si vous débutez.

Si vous souhaitez construire une requête depuis zéro, veuillez consulter notre guide sur [l’autorisation d’une requête OAuth 1.0a](/fr/resources/fundamentals/authentication/oauth-1-0a/authorizing-a-request).
